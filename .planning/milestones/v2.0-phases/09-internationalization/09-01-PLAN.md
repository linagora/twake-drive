4. Do NOT change LANG_NAMES — these stay as native language names per user decision.

**scribeAI.js changes:**

1. Refactor `deriveLoadingMessage(actionId, label)` to `deriveLoadingMessage(actionId, label)` that returns an i18n key (or an object `{ key, params }` for interpolated messages) instead of English strings:
   - `free-prompt` -> return `{ key: 'Scribe.loading.processing' }`
   - `translate-*` -> return `{ key: 'Scribe.translate.translating_to', params: { language: label } }`
   - Known action IDs -> return `{ key: LOADING_KEYS[actionId] }` via a `LOADING_KEYS` map:
     ```
     'correct-grammar': 'Scribe.loading.correct_grammar',
     'tone-professional': 'Scribe.loading.tone_professional',
     'tone-casual': 'Scribe.loading.tone_casual',
     'tone-polite': 'Scribe.loading.tone_polite',
     'improve-shorter': 'Scribe.loading.improve_shorter',
     'improve-expand': 'Scribe.loading.improve_expand',
     'improve-emojify': 'Scribe.loading.improve_emojify',
     'improve-bullets': 'Scribe.loading.improve_bullets'
     ```
   - Fallback for unknown IDs -> return `{ key: 'Scribe.loading.processing' }`

2. Refactor `classifyScribeError(err)` to return `{ messageKey, canRetry }` instead of `{ message, canRetry }`:
   - AbortError -> `{ messageKey: '', canRetry: false }`
   - 401/403 -> `{ messageKey: 'Scribe.error.auth', canRetry: false }`
   - 429 -> `{ messageKey: 'Scribe.error.rate_limit', canRetry: true }`
   - 500+ -> `{ messageKey: 'Scribe.error.server', canRetry: true }`
   - Other FetchError -> `{ messageKey: 'Scribe.error.generic', canRetry: false }`
   - TypeError/network -> `{ messageKey: 'Scribe.error.network', canRetry: true }`
   - Empty response -> `{ messageKey: 'Scribe.error.empty_response', canRetry: true }`
   - Default -> `{ messageKey: 'Scribe.error.unexpected', canRetry: true }`

3. Update JSDoc comments to reflect the new return types. Update the exports comment at the top.

4. Do NOT change `buildMessages`, `callScribeAI`, `SYSTEM_PROMPT`, or `findActionConfig` — those stay as-is.
