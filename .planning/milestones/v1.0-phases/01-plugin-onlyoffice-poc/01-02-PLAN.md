Selection detection (automatic via config):
  - config.json: initOnSelectionChanged: true, initDataType: "text"
  - plugin.init(data) called on every selection change

  executeMethod("GetSelectedText", [{

    TableCellSeparator: "\n",

Replace selected text:
  executeMethod("PasteText", ["replacement text"])

  Asc.scope.textToInsert = text;
  callCommand(function() {
    var oDocument = Api.GetDocument();
    var oParagraph = Api.CreateParagraph();
    oParagraph.AddText(Asc.scope.textToInsert);
    oDocument.InsertContent([oParagraph]);
  }, false, false, callback)

  ```javascript
  function replaceWithMock() {
    if (!lastSelectedText) { log("No selection to replace"); return; }
    var transformed = mockTransform(lastSelectedText);
    log("Replacing selection with mock transform...");
    window.Asc.plugin.executeMethod("PasteText", [transformed]);
  }
  ```
  - PasteText replaces the current selection with the transformed text
  - After PasteText completes, log success
  - If no text is selected, log a warning and do nothing

  **PLUG-05: Insert text after selection with line break**
  Wire the "Insert After" button to:
  ```javascript
  function insertAfterSelection() {
    if (!lastSelectedText) { log("No selection for insert"); return; }
    var transformed = mockTransform(lastSelectedText);
    log("Inserting mock transform after selection...");
    Asc.scope.textToInsert = transformed;
    window.Asc.plugin.callCommand(function() {
      var oDocument = Api.GetDocument();
      var oParagraph = Api.CreateParagraph();
      oParagraph.AddText(Asc.scope.textToInsert);
      oDocument.InsertContent([oParagraph]);
    }, false, false, function(result) {
      // Callback after callCommand completes
      // Note: cannot reference plugin variables here
    });
  }
  ```

  **CRITICAL callCommand constraints:**
  - `false, false` parameters: first false = do not rewrite the paste format, second false = do not use recalc (synchronous execution)
  - ALL data must be in `Asc.scope` before the call
  - The function body runs in an ISOLATED sandbox -- no `lastSelectedText`, no `log()`, no plugin variables
  - Only `Api.*` and `Asc.scope.*` are available inside
  - The callback (4th parameter) runs back in plugin context

  **InsertContent behavior testing:**
  Per RESEARCH.md Open Question 2: it is unclear whether `InsertContent` inserts AFTER the selection or replaces it. The executor must test this. If `InsertContent` replaces the selection instead of appending:
  - Workaround: Before calling `callCommand`, first deselect and move cursor to end. This may require: `Asc.scope.originalText = lastSelectedText; Asc.scope.textToInsert = transformed;` then inside callCommand, use the cursor positioning API to move to end of selection before inserting.
  - Document the actual behavior in README.md regardless of outcome.

  **SearchAndReplace fallback (selection-loss workaround):**
  Per RESEARCH.md Pitfall 3, if the user clicks in the test panel (which IS inside the plugin iframe), focus may shift away from the editor and the selection could be lost. Since the test panel is inside the same plugin iframe, this may or may not happen. If PasteText replaces at the wrong position because selection was lost:
  - Add a fallback using `SearchAndReplace`: `executeMethod("SearchAndReplace", [{searchString: lastSelectedText, replaceString: transformed, matchCase: true}])`
  - Only use as fallback -- try PasteText first since it's simpler
  - Document the behavior in README.md
