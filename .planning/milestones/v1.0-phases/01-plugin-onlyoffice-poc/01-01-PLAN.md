  <action>
  Create the `plugins/onlyoffice-scribe/` directory with the standard OnlyOffice plugin structure:

  **config.json:** Use the exact config from RESEARCH.md Pattern 1:
  - `"name": "Scribe"`
  - `"guid": "asc.{C36B0B33-0C65-4E88-9EE0-C1D6A40434EC}"`
  - `"version": "0.1.0"`, `"minVersion": "8.2.0"`
  - Single variation: `type: "background"`, `initDataType: "text"`, `initOnSelectionChanged: true`
  - `EditorsSupport: ["word"]`, `buttons: []`
  - `events: ["onContextMenuShow", "onContextMenuClick"]`

  **index.html:** Minimal HTML that loads the OO Plugin SDK from CDN (`https://onlyoffice.github.io/sdkjs-plugins/v1/plugins.js`) and then `scripts/code.js`. Use the exact structure from RESEARCH.md.

  **scripts/code.js:** Minimal skeleton that ONLY does:
  - `window.Asc.plugin.init = function(data) { console.log("[Scribe] Plugin loaded. Selection:", data ? data.substring(0, 80) : "(none)"); }`
  - `window.Asc.plugin.button = function(id) { this.executeCommand("close", ""); }`
  - Wrap in IIFE: `(function(window, undefined) { "use strict"; ... })(window, undefined);`
  - This is the bare minimum to confirm the plugin loads. Selection detection, text reading, replacement, and insertion come in Plan 02.

  **Icon files:** Create minimal 40x40 PNG placeholder icons for `resources/light/icon.png` and `resources/dark/icon.png`. These can be simple single-color squares (the icon format is required by OO but content doesn't matter for POC). Use a simple approach: either create them with ImageMagick (`convert -size 40x40 xc:blue icon.png`) or use a base64-encoded minimal PNG written via a script.

  **scripts/oo-dev-setup.sh:** Create a bash script that:
  1. Prints usage instructions
  2. Runs `docker run -itd -p 80:80 --name oo-dev -v "$(pwd)/plugins/onlyoffice-scribe:/var/www/onlyoffice/documentserver/sdkjs-plugins/scribe" onlyoffice/documentserver` (per user decision: standalone docker run, NOT docker-compose)
  3. Waits for container to be ready
  4. Runs `docker exec oo-dev dpkg -l onlyoffice-documentserver` to print the OO version
  5. Runs the plugin registration command: `docker exec oo-dev bash -c '/usr/bin/documentserver-pluginsmanager.sh --directory="/var/www/onlyoffice/documentserver/sdkjs-plugins" --update="/var/www/onlyoffice/documentserver/sdkjs-plugins/plugin-list-default.json"'` (or equivalent to register the plugin)
  6. Prints a message: "Setup complete. Open http://localhost in your browser."
  7. Make it executable with `chmod +x`

  Per user decision: Docker config changes are documented only, don't modify existing Docker config files in the repo. This script is a NEW dev helper, not modifying existing infra.

  IMPORTANT: Mount ONLY the scribe plugin subfolder, NOT the entire `sdkjs-plugins/` directory (per Pitfall 4 in RESEARCH.md -- mounting the parent would hide all default plugins).
  </action>
