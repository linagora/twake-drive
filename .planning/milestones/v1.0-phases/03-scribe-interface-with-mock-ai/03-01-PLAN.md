Props:
- `breadcrumb` (string): Path taken, e.g. "Translate > Anglais"

- `onInsert` (function): Called when Insert is clicked

Structure (per CONTEXT.md decisions):
```
+------------------------------------------+
| [breadcrumb text]              [X close]  |
+------------------------------------------+

|                                           |
|                                           |

|                    Replace    [Inserer]    |
+------------------------------------------+
```

- Header: breadcrumb text (Typography, bold or caption) + close button (IconButton with Cross icon) aligned right
- Result text area: read-only div with `whiteSpace: pre-wrap`, scrollable if content overflows (maxHeight ~300px)
- Bottom actions: "Replace" as `Buttons variant="text"` (blue text) and "Inserer" as `Buttons` default/primary (blue pill) -- aligned bottom-right
- Use cozy-ui Typography, Buttons, Icon (Cross), Paper
- Min-width: 280px for readability
- Include PropTypes

**ScribePopover.jsx** -- Main popover container managing the two-step state machine.

Props (same interface as ScribeModal for drop-in replacement):
- `open` (boolean): Whether the popover is visible
- `selectedText` (string): Text selected in the editor
- `onReplace` (function): Called with transformed text
- `onInsert` (function): Called with transformed text
- `onCancel` (function): Called when closed without action

State:
- `step`: `'menu'` | `'result'` -- which step to show
- `result`: `{ text: '', breadcrumb: '' }` -- transformation result

Behavior:
- When `open` transitions to true, reset `step` to `'menu'` and clear result (useEffect watching `open`)
- `handleActionSelect(actionId, label, breadcrumb)`: calls `mockTransform(actionId, selectedText)`, stores result, sets step to `'result'`
- `handleClose`: resets state, calls `onCancel()`
- `handleReplace`: calls `onReplace(result.text)`, resets state
- `handleInsert`: calls `onInsert(result.text)`, resets state
- Use `useCallback` for all handlers, `useMemo` where appropriate

Rendering:
- Use `cozy-ui/transpiled/react/Popover` with `anchorReference="anchorPosition"` and `anchorPosition={{ top: 200, left: window.innerWidth / 2 }}`
- `BackdropProps={{ invisible: true }}` -- no dimmed backdrop (per CONTEXT.md)
- `PaperProps={{ style: { borderRadius: 8, boxShadow: '0 4px 20px rgba(0,0,0,0.15)' } }}` -- subtle shadow
- Conditionally render based on `step`:
  - `'menu'`: Render a placeholder div with text "Action menu (Plan 03-02)" -- Plan 03-02 will replace this with ScribeActionMenu
  - `'result'`: Render `<ScribeResultPanel>` with result data and handlers
- Export as named export: `export { ScribePopover }`
- Include PropTypes

IMPORTANT: The popover currently renders a placeholder for the menu step. Plan 03-02 will create ScribeActionMenu.jsx and ScribePromptInput.jsx, and update ScribePopover.jsx to import and render them. This is the interface-first approach -- ScribePopover defines the container and state machine, Plan 03-02 fills in the menu content.
