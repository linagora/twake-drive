**useCozyBridge.js** -- React hook wrapping CozyBridge lifecycle:

```javascript
import { useEffect, useState, useCallback, useRef } from 'react'
import { CozyBridge } from '@/lib/cozy-bridge'
```

The hook:
- Takes `allowedOrigins` (array of strings) as parameter
- Creates a `CozyBridge` instance in a `useEffect`, stores it in a `useRef`
- Registers a handler for `'AI_TEXT_EDIT'` intent that:
  - Stores the received `intentMessage` in state (`pendingIntent`)
  - Stores the `respondFn` in a ref (`respondRef`)
- Exposes a `respond(responsePayload)` callback that:
  - Calls `respondRef.current(responsePayload)` to send the response back
  - Sets `pendingIntent` to null (closes modal)
  - If `respondRef.current` is null, logs warning
- Returns `{ pendingIntent, respond }` -- same interface as the research code example
- Cleanup: calls `bridge.destroy()` in the useEffect cleanup function
- The `allowedOrigins` parameter should be stable (useMemo or passed from parent). Include a comment: `// In dev: ['http://localhost']. In prod: derive from instance URL and OO server URL`

**For the allowed origins:** In development, use a permissive approach. The hook should accept any origin during dev (pass `['*']` or handle `'*'` as a wildcard in the CozyBridge class -- check if CozyBridge already handles this, if not, add a special case: if allowedOrigins includes `'*'`, skip origin check). For production, origins should come from the Cozy instance URL and the OO server URL.

**Important:** The `allowedOrigins` array comparison in useEffect deps will cause re-renders. Either: (a) accept `allowedOrigins` as a ref/stable value, or (b) serialize to string for comparison. Use approach (a): accept that the parent passes a stable array (useMemo'd).
