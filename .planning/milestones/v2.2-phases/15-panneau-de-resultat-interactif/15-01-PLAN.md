  <name>Task 1: Add drag-to-move on result panel background</name>
  <files>src/modules/views/OnlyOffice/Scribe/ScribePopover.jsx, src/modules/views/OnlyOffice/Scribe/ScribeResultPanel.jsx</files>
  <action>
In ScribePopover.jsx:
- Add state: `const [dragOffset, setDragOffset] = useState({ x: 0, y: 0 })` -- represents pixel offset from center position.
- Reset dragOffset to `{ x: 0, y: 0 }` in the existing `useEffect` that fires when `open` changes (alongside the existing step/result resets).
- Modify `anchorPosition` to incorporate dragOffset: `{ top: window.innerHeight / 2 + dragOffset.y, left: window.innerWidth / 2 + dragOffset.x }`.
- Pass `onDragMove={setDragOffset}` and `dragOffset={dragOffset}` as props to ScribeResultPanel.

In ScribeResultPanel.jsx:
- Accept new props: `onDragMove` (function) and `dragOffset` ({ x, y }).
- Add a `useRef` for tracking drag state: `dragStateRef = useRef({ dragging: false, startX: 0, startY: 0, startOffsetX: 0, startOffsetY: 0 })`.
- On the outer Paper element, add `onMouseDown={handleDragStart}`.
- `handleDragStart(e)`: Only start drag if `e.target === e.currentTarget` OR if the target is the header area (breadcrumb Typography) but NOT a button/IconButton/input. Check by walking up from `e.target` to the Paper -- if any ancestor is a `button`, `a`, `input`, `textarea`, or has `role="button"`, or has class containing `scribe-result-text` or `scribe-result-actions`, do NOT start drag. Otherwise, set `dragStateRef.current = { dragging: true, startX: e.clientX, startY: e.clientY, startOffsetX: dragOffset.x, startOffsetY: dragOffset.y }`, add `document.addEventListener('mousemove', handleDragMove)` and `document.addEventListener('mouseup', handleDragEnd)`, and set `e.preventDefault()` to prevent text selection.
- `handleDragMove(e)`: If dragging, compute `dx = e.clientX - dragState.startX`, `dy = e.clientY - dragState.startY`, call `onDragMove({ x: dragState.startOffsetX + dx, y: dragState.startOffsetY + dy })`.
- `handleDragEnd()`: Set dragging=false, remove document listeners.
- Cleanup: In a useEffect return, remove any lingering document listeners on unmount.
- Add `cursor: 'move'` on the Paper element (but buttons/text areas will override with their own cursor via CSS specificity).
- Update PropTypes to include onDragMove and dragOffset.

IMPORTANT: The drag must NOT interfere with:
- The existing onKeyDown focus-trap handler on Paper
- Clicking buttons (Insert, Replace, Close, Retry)
- Scrolling the result text content area
- Text selection within the result text area (class scribe-result-text)
  </action>

  <files>src/modules/views/OnlyOffice/Scribe/ScribePopover.jsx, src/modules/views/OnlyOffice/Scribe/ScribeResultPanel.jsx, src/modules/views/OnlyOffice/Scribe/scribe.styl</files>
  <action>
In ScribePopover.jsx:
- Add state: `const [panelSize, setPanelSize] = useState(null)` -- null means "use default CSS sizing", otherwise `{ width, height }` in pixels.
- Reset panelSize to `null` in the same `useEffect` that resets on `open`.
- Pass `panelSize={panelSize}` and `onResize={setPanelSize}` as props to ScribeResultPanel.

In ScribeResultPanel.jsx:
- Accept new props: `panelSize` (null or { width, height }) and `onResize` (function).
- Apply panelSize to the outer Paper element via inline style: if panelSize is not null, set `style={{ width: panelSize.width, height: panelSize.height, maxWidth: '95vw', maxHeight: '90vh' }}` and also add `display: 'flex', flexDirection: 'column'` so that the content area (scribe-result-text) flexes to fill available space. The result text div should get `flex: 1, minHeight: 0` when panelSize is set so it grows/shrinks with the panel.
- Add a resize handle div as the LAST child inside Paper (after scribe-result-actions), with class `scribe-resize-handle`.
- The resize handle: `onMouseDown={handleResizeStart}` with `e.stopPropagation()` (prevent drag-to-move from firing). Track resize state in a ref similar to drag: `resizeStateRef = useRef({ resizing: false, startX, startY, startWidth, startHeight })`.
- `handleResizeStart(e)`: Get current panel dimensions from Paper ref (`paperRef.current.getBoundingClientRect()`). Store start mouse position and start dimensions. Add document mousemove/mouseup listeners. Prevent default.
- `handleResizeMove(e)`: Compute new width = startWidth + (e.clientX - startX), new height = startHeight + (e.clientY - startY). Clamp: min 250px width, min 150px height, max 95vw width, max 90vh height. Call `onResize({ width: newWidth, height: newHeight })`.
- `handleResizeEnd()`: Set resizing=false, remove listeners.
- Add `paperRef = useRef(null)` and put `ref={paperRef}` on the Paper element (Paper from cozy-ui supports ref forwarding).
- Cleanup in useEffect return for unmount.
- Update PropTypes.

In scribe.styl:
- Add `.scribe-resize-handle` style:
  - `position absolute` (Paper needs `position relative`)
  - `bottom 0`, `right 0`
  - `width 16px`, `height 16px`
  - `cursor nwse-resize`
  - `opacity 0.3` (subtle), `opacity 0.6` on hover
  - `transition opacity 150ms ease`
  - Use a CSS triangle or border trick for visual indicator: `border-right 2px solid currentColor`, `border-bottom 2px solid currentColor` repeated with small offsets, or simply use `background` with a small grip pattern. Simplest: two diagonal lines using a pseudo-element or a background-image SVG data URI showing the classic resize grip (three diagonal parallel lines).
  - Recommendation: Use `&::after` pseudo-element with content `''`, a small triangle or lines in bottom-right. Example: `background-image url("data:image/svg+xml,...")` with a 3-line grip icon in currentColor.
- Add `position relative` to `.scribe-result-panel` (needed for absolute-positioned resize handle).

IMPORTANT: The resize handle must NOT interfere with the focus trap keyboard navigation. It should not be in the Tab order (no tabIndex, or tabIndex=-1).
