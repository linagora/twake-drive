  <name>Task 1: Suppress menu item highlight until mouse physically moves</name>
  <files>src/modules/views/OnlyOffice/Scribe/ScribeActionMenu.jsx</files>
  <action>
Add a `mouseMoveEnabled` ref (useRef(false)) to ScribeActionMenu. This ref gates whether onMouseEnter on menu items should update focusIndex.

1. Add `const mouseMoveEnabledRef = useRef(false)` near the top of the component.

2. Add a `useEffect` that resets `mouseMoveEnabledRef.current = false` whenever the component mounts/re-renders fresh (this handles the menu opening case). Use the `paperRef` to attach a one-time `mousemove` listener that sets `mouseMoveEnabledRef.current = true` after detecting actual mouse movement. The key insight: a `mousemove` event only fires when the mouse physically moves, not when a menu opens under a stationary cursor.

Specifically:
```javascript
useEffect(() => {
  mouseMoveEnabledRef.current = false
  const paper = paperRef.current
  if (!paper) return

  const enableMouse = () => {
    mouseMoveEnabledRef.current = true
    paper.removeEventListener('mousemove', enableMouse)
  }
  paper.addEventListener('mousemove', enableMouse)

  return () => {
    paper.removeEventListener('mousemove', enableMouse)
  }
}, [])
```

3. In every `onMouseEnter` handler on ListItem elements (both main menu items at ~line 253 and submenu items), guard with `if (!mouseMoveEnabledRef.current) return` as the first line.

4. Also guard the `onMouseLeave` handler at ~line 257 that closes submenus: `if (!mouseMoveEnabledRef.current) return`.

5. Also guard the prompt Paper `onMouseEnter` at ~line 385: `if (!mouseMoveEnabledRef.current) return`.

This ensures that when the Popover opens and the cursor happens to be over a menu item, no highlight occurs until the user physically moves the mouse.
  </action>

6. Add a cleanup `useEffect` that clears the timer on unmount: `useEffect(() => () => clearTimeout(timerRef.current), [])`.
7. Change the tooltip render condition from `{hovered && (` to `{showTooltip && (`.

This way: hover makes button opaque instantly, but the tooltip text only appears after 1 full second of continuous hover. Moving away clears the timer, preventing tooltip from appearing.
