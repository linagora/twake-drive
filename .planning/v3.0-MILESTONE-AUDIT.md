## Integration Bugs

### Bug 1: Use-Before-Define on panelActions handlers

**File:** `src/modules/views/OnlyOffice/View.jsx`
**Lines:** useEffect at 100-104, useCallbacks at 132-148

The `useEffect` that calls `setPanelActions({ replace: handleReplace, insert: handleInsert })` is defined before the `useCallback` declarations for `handleReplace` and `handleInsert`. On initial render, the effect closure captures `undefined` for both handlers. The effect will re-run when `respond` or `focusEditor` identity changes, but there's a narrow window where Insert/Replace clicks silently no-op.

**Fix:** Move `handleReplace` and `handleInsert` `useCallback` declarations above the `setPanelActions` useEffect, or use a ref pattern (like the existing `handleCancelRef`).

**Affected:** ACT-02, ACT-03

### Bug 2: SHOW_SCRIBE_BUTTON missing HTML payload

**File:** `useCozyBridge.js` line 55, `View.jsx` line 85

The `SHOW_SCRIBE_BUTTON` intent only carries `{ text }`, not `{ text, html }`. When the panel selection chip is set via this path (floating button click), the AI receives plain text without rich formatting. The `pendingIntent` path correctly carries HTML.

**Impact:** Minor — SEL-02 fully met on popover path, plain-text fallback on floating button path.
