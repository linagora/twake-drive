- All 5 locales now report 46 `Scribe.*` leaf keys.

**Task 2 — parity gate (commit 281a67a26):**
- `src/locales/scribeI18nParity.spec.js`: imports fr/en/de/es/it JSON directly, flattens the `Scribe` namespace to sorted dotted leaf keys, and asserts en/de/es/it deep-equal the fr reference set. On drift it prints the locale plus the symmetric difference (missing/extra). A second block asserts every leaf is a non-empty trimmed string across all 5 locales.
- Scoped to the 5 target locales only (non-target ar/ja/ko/nl/nl_NL/pl/ru/zh_* are explicitly out of scope per CONTEXT and would falsely fail).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Italian JSON import shadowed Jest's `it` global**
- **Found during:** Task 2 (first spec run)
- **Issue:** `import it from './it.json'` overrode the global `it` test function → `TypeError: (0 , _itjson.default) is not a function`, suite failed to run.
- **Fix:** Renamed all locale imports to `localeFr/localeEn/localeDe/localeEs/localeIt` and mapped them in the `LOCALES` registry. Added an explanatory comment.
- **Files modified:** src/locales/scribeI18nParity.spec.js
- **Commit:** 281a67a26

**2. [Rule 1 - Bug] Vitest-style `expect(value, message)` is silently ignored by Jest**
- **Found during:** Task 2 (authoring)
- **Issue:** The two-arg `expect(received, message)` signature is a Vitest API; under Jest the message arg is ignored, so the intended self-explaining failure text would never surface.
- **Fix:** Replaced with labeled object-equality assertions (`expect({locale, missing, extra}).toEqual({locale, missing: [], extra: []})`) so a failure still prints the locale and the exact diverging keys via Jest's standard diff.
- **Files modified:** src/locales/scribeI18nParity.spec.js
- **Commit:** 281a67a26
