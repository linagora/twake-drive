Contract shape (SCRIBE_OUTPUT_SCHEMA):
  { "discussion": string (required), "fragments": string[] (optional, defaults to []) }
  - `discussion`: conversational markdown; may embed `{{fragment:N}}` position markers (0-indexed)
  - `fragments[N]` corresponds to marker `{{fragment:N}}`
  - Rich markers ([TABLE:N]…[/TABLE], [CELL:r,c]…[/CELL], [^scribe-fn-N], {{REF:scribe-ref-N:…}})
    belong INSIDE fragment strings, NEVER in discussion.

Existing buildMessages contract (scribeAI.js) — PRESERVE:
  - Single user-role message (no system role) for RAG-backend compatibility
  - systemBase = SYSTEM_PROMPT, conditionally extended when enrichedMd contains
    '[TABLE:' / '[CELL:' / '[^scribe-fn-' / '{{REF:'
  - systemPrefix = systemBase + '\n\n', then prepended to the action/free-prompt body
