# Phase v3.1-02 Plan 03: Inline parse / normalize / mirror Summary

Wired the v3.1-01 response contract through the INLINE popover surface: `handleActionSelect` now parses the raw LLM response with `parseScribeResponse(text, { surface: 'popover' })` before any display or storage, normalizes the parsed result to exactly one insertable fragment, shows/inserts only that normalized fragment (discussion not rendered), and mirrors the full parsed turn into shared chat history using plan 02's extended message model. Completes the inline half of the zero-regression proof (INLINE-01, INLINE-02).

## What Changed

### Task 1 — Inline parse seam + single-fragment normalization + normalized-fragment display/insert (commit 29d4fbedf)
- Added `import { parseScribeResponse } from '.../scribeResponse'`.
- In `handleActionSelect` success branch, immediately after `callScribeAI`:
  - **D-13:** `const parsed = parseScribeResponse(text, { surface: 'popover' })` runs BEFORE `transformCellMarkersForPreview` / `setRawResult` / `setResult` / `addMessage`.
  - **D-08:** computed `normalizedFragment` via an explicit if/else chain — `fragments.length >= 2` → `fragments.join('\n\n')`; `=== 0` → `parsed.discussion`; else `fragments[0]`.
  - **D-09:** `transformCellMarkersForPreview(normalizedFragment, enrichedMd)` and `setRawResult(normalizedFragment)` now operate on the normalized fragment (replacing the raw `text`). `discussion` is not passed to `setResult` / not rendered in the popover. `cellWarning` and `result.text` (displayMd) wiring otherwise unchanged.
  - `handleReplace`/`handleInsert` still resolve to `rawResult` (now the normalized fragment) — unchanged.
- The test-markdown dev bypass, error branch, and abort logic were left untouched and are NOT routed through `parseScribeResponse`.

### Task 2 — Mirror the full parsed inline turn into shared chat history (commit 7f435fb41)
- **D-11:** Success-branch assistant `addMessage` turn now carries `content: parsed.discussion`, plus `discussion`, `fragments`, `fellBack` (matching plan 02's extended model), instead of the verbatim `content: text`.
- Reuses the `parsed` object from Task 1 — no second `parseScribeResponse` call.
- The user breadcrumb turn (`role: 'user', content: breadcrumb`) is preserved; the error-branch mirror (`role: 'error'`) is unchanged.
- Mirrored assistant content is the discussion channel, NOT the normalized single-fragment blob — so the chat render-time compose helper (ChatMessageList) renders inline turns identically to chat-native turns, and v3.1-04 cards will light up for inline turns too.

## No-Regression Proof

On the popover fallback path (`surface: 'popover'` yields `fragments == [raw]`), normalization takes the 1-fragment branch and returns `raw` unchanged as the single fragment. `transformCellMarkersForPreview` and `rawResult` therefore receive byte-identical input to today, so Insert/Replace produce byte-identical insertable text. The only new behavior on the success path is routing already-received text through the tested parser and selecting one normalized fragment.
