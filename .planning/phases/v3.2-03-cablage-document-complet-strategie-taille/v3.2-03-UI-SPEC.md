# Phase v3.2-03 — UI Design Contract

> Visual and interaction contract for the document-truncation feedback surface of the
> "Câblage document complet + stratégie de taille" phase. Generated by gsd-ui-researcher,
> verified by gsd-ui-checker.

This is a **wiring phase with one small, edge-case UI surface**. The « document » checkbox
already shipped (v3.2-01, `ScribeIncludeZone.jsx`). The only NEW visual surfaces this phase
introduces are:

1. **D-03 — the "document truncated" feedback notice** (primary UI deliverable).
2. **The extraction-pending feedback decision** (D-05) — resolved here as **REUSE** of the
   existing chat loading state (no new pending UI).
3. **A secondary "document could not be read" failure notice** (the other half of the
   "never silent" guarantee, CTX-LLM-04 #3 / D-03) — resolved as REUSE of the existing
   `ErrorBubble` family.

No new prompt-composition UI, no new checkboxes, nothing beyond truncation / extraction
feedback. The rest of the phase is non-visual wiring at the `ScribeContext.sendMessage` seam.

---

## Locked Design Decisions (phase-specific)

These are the load-bearing visual/interaction decisions the orchestrator explicitly delegated
to this UI-SPEC. All are grounded in the existing Scribe panel design system — no new visual
language is introduced.

### DEC-UI-01 — Truncation indicator: **after-send, inline system notice in the chat thread** (NOT a persistent badge in the Inclure zone)

- **Form:** a discreet, single-line **system notice** rendered **inside the chat thread**, in
  document order, **directly after the user message bubble** of the turn whose document
  context was truncated. It is **NOT a chat bubble** and **NOT** a badge attached to the
  « document » checkbox.
- **Timing:** **after send**, not a before-send preview/estimate. Rationale (all from CONTEXT):
  - D-02 makes the default **unlimited** — in the common case **nothing truncates**, so a
    persistent before-send badge/estimate would clutter the always-visible Inclure zone for
    an event that rarely fires. The phase scope explicitly warns *"keep it proportionate and
    discreet, do not let it clutter the panel."*
  - D-05 forces a **fresh extraction on every send**; whether truncation happens is only
    known **at send time** (extract → estimate vs configured budget → maybe cut). The honest,
    proportionate moment to inform the user is **when it actually happens**, anchored to the
    turn that triggered it.
  - The document can change between turns (D-05, no cache), so a persistent steady-state badge
    in the Inclure zone would misrepresent a per-send event.
- **Why NOT the Inclure-zone badge option:** the Inclure zone (`ScribeIncludeZone.jsx`) was
  deliberately tightened to be the quietest element on the panel (10/11px, `text.disabled`,
  fixed no-reflow slot). Hanging a conditional truncation badge off the « document » checkbox
  would re-introduce reflow risk and add weight to the one zone the team explicitly de-emphasized.
- **Visual treatment:** a **sibling of the existing loading indicator** (`ChatMessageList.jsx`
  l.519-534) — `display:flex; alignItems:center; gap:8px; padding:8px 12px;` left-aligned
  (`alignSelf:flex-start`), with a small **info icon** + muted text. See Component Inventory.
- **"Discreet ≠ hidden":** explicitness (CTX-LLM-04 #3, "never silent") is carried by the
  **words**, not by loud color. The notice is a persistent text line in the thread — the user
  is told plainly; it simply does not shout.

### DEC-UI-02 — Extraction-pending feedback: **REUSE the existing chat loading state** (no new pending UI)

- D-05's fresh-extraction round-trip happens **inside the existing send→response window**:
  `sendMessage` already sets `isLoading=true`, which renders the existing thinking indicator
  (`Spinner` + `t('Scribe.chat.typing')` = "Scribe is thinking…", `ChatMessageList.jsx` l.519).
  The extraction latency on large docs is absorbed by this same window.
- **No distinct "extracting document…" hint is warranted.** Adding a second pending state for
  an edge (large docs only) adds complexity for marginal value and conflicts with the
  proportionate/discreet directive.
- The existing label `Scribe.chat.typing` ("Scribe is thinking…") is **phase-agnostic** and
  remains accurate while extraction runs. **Optional planner discretion:** if a turn has
  `includeDocument` checked, the planner MAY swap the loading caption to a document-aware
  i18n string (key reserved below as `Scribe.loading.reading_document`) — this is **optional
  polish, not required** by this contract. Default = reuse `Scribe.chat.typing` unchanged.

### DEC-UI-03 — Extraction failure: **REUSE the existing `ErrorBubble`** (the other "never silent" path)

- If the document **cannot be extracted** (plugin error / no response) or the extraction
  **exceeds the 1 MB bridge transport cap** in a way that cannot be safely bounded into a
  truncation, the message must still be sent **without** the document, and the user must be
  **told it was not included** (never silent).
- Treatment: the existing `ErrorBubble` in the thread (`ChatMessageList.jsx` l.439-447) — the
  established error family — with a new i18n string (reserved below). This is **distinct** from
  DEC-UI-01: truncation is *informational* (part of the doc was sent); failure is an *error*
  (none of the doc was sent). Different tone, different component.
- This stays within the phase scope ("truncation / extraction feedback") and closes the
  failure branch of the "never silent" guarantee.

### DEC-UI-04 — Theme-aware, no new tokens

- All three surfaces resolve colors through the existing `useTheme()` palette (light/dark),
  exactly like `SelectionChip.jsx`, `ScribeIncludeZone.jsx`, and the loading indicator. No new
  hex constants are introduced by this phase (the brand `#7C3AED` accent is **not** used by any
  of these surfaces — see Color).

## Spacing Scale

Declared from **actual** Scribe-panel usage (all multiples of 4):

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Inner inline padding; chip vertical padding (`SelectionChip` `4px 0`) |
| sm | 8px | Icon↔text gap in the notice; row gap; vertical padding of the notice line (`8px 12px`) |
| md | 12px | Horizontal padding of thread rows / notice line (`8px 12px` — matches loading indicator & include zone) |
| lg | 16px | Thread container padding (`ChatMessageList` `padding:16`) |
| xl | 24px | Reserved (chip body height region) |
| 2xl | 32px | Reserved (Send button footprint, reserved chip slot height) |

Exceptions: none. (The codebase uses a 4/8/12/16/24/32 step set; 12px is heavily used for
horizontal row padding and is a multiple of 4. This phase's notice MUST reuse `8px 12px` to
sit flush with the existing loading indicator.)

as a sibling of the loading indicator.

|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Chat content, assistant bubbles, prompt textarea, `ErrorBubble` (extraction-failure path) |

uses **400** (it is not a heading and must not shout).

(muted) for both icon and label, matching the loading indicator's quiet register. Explicitness
comes from the copy, not color loudness.

## Interaction Contract

| Trigger | Condition | Behavior |
|---------|-----------|----------|
| User sends a message with « document » checked, default config (no budget) | D-02 unlimited → never truncates | Full document injected; **no notice shown** (silent success is correct here — nothing was cut) |
| User sends with « document » checked, configured budget exceeded | Truncation occurs at the planner-defined cut point | **Truncation notice (DEC-UI-01)** appears in the thread after the user bubble for that turn; persists in the transcript |
| Document extraction in flight (any large doc) | `isLoading === true` during the send→response window | Existing thinking indicator (`Spinner` + `Scribe.chat.typing`); **no new pending UI** |
| Extraction fails / unrecoverable transport-cap overflow | Document could not be included at all | Message still sent **without** the document; **`ErrorBubble` notice (DEC-UI-03)** tells the user it was not included (never silent) |
| « document » unchecked | — | No extraction, no notice, no change to the loading indicator |

A11y: the truncation notice uses `role="status"` + `aria-live="polite"` so it is announced when
it appears, without stealing focus. It is text + icon (icon is decorative; the text carries the
meaning, so no `aria-label` on the icon is required).

The notice is **per-turn and persistent** in the transcript (it stays attached to the turn that
triggered it, scrolls with history) — it is not a transient toast.

## Copywriting Contract

This phase has **no new primary CTA** (the Send button predates this phase). Copy below is the
new user-facing text; all strings go through `t()` — **zero hard-coded strings** (v3.1-07
literal-audit + 5-locale parity gates are active for fr/en/de/es/it).

| Element | Copy (EN canonical) |
|---------|---------------------|
| Primary CTA | n/a — no new CTA this phase (existing Send button unchanged) |
| Empty state | n/a — no new empty surface (chat `WelcomeMessage` unchanged) |
| Truncation notice (DEC-UI-01) | "Document too long — only part of it was sent to the assistant." |
| Extraction-failure notice (DEC-UI-03) | "Couldn't read the document — your message was sent without it." |
| Loading (reuse) | existing "Scribe is thinking…" (`Scribe.chat.typing`) — unchanged |
| Optional doc-aware loading (DEC-UI-02, not required) | "Reading the document…" (key reserved, off by default) |
| Destructive confirmation | none — this phase has no destructive actions |

Copy guidance for the planner:
- The truncation copy is **cut-point-agnostic on purpose** ("only part of it"), because the
  cut algorithm (from start / around the selection / reserving room for the answer) is
  **planner discretion** (D-02 / Claude's Discretion). Do **not** assert "the beginning" or a
  specific fraction unless the cut algorithm is finalized to match.
- Keep both notices to a **single short line** (Caption role) — explicit but discreet.

## Out of Scope (do NOT design here)

- Any new prompt-composition UI, new checkboxes, or changes to the Inclure zone layout.
- A persistent truncation badge on the « document » checkbox (explicitly rejected — DEC-UI-01).
- A distinct "extracting…" pending spinner separate from the existing loading state
  (explicitly rejected — DEC-UI-02; optional label swap only).
- The cut-point algorithm, the size budget config shape/unit, the extraction intent name, and
  the 1 MB transport-cap handling — these are **non-visual wiring / planner discretion** (CONTEXT
  D-01/D-02 + Claude's Discretion). This contract only governs what the user *sees* when those
  fire.
