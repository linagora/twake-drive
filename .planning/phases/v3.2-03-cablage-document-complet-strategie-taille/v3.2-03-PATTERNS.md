### `plugins/onlyoffice-scribe/scripts/code.js` — new full-document extractor + listener (plugin, ES5)

**Analog A — document-order walk:** `hookDumpState` (`code.js:3225-3344`). This is the **correct primitive** for whole-document iteration. Do NOT use `doc.GetAllParagraphs()` (it silently omits table-cell paragraphs — confirmed by the codebase's own compensating comment at `code.js:638`).

```javascript

  if (ct === "paragraph") {
    var idx = blocks.length;

  } else if (ct === "table") {
    blocks.push(tableToBlock(el));
  }
}
```
The callCommand here is invoked **read-only** (`}, false, false, function(ret){...}` at `code.js:3340`) — confirm the read-only semantics for the new extractor too (see Open Question 1 / Pitfall 5 in RESEARCH).

**Analog B — leaf emitters to REUSE (do not rebuild):** all live INSIDE the init `callCommand` body (the ES5 sandbox, `code.js:2072-2936`):
- `paragraphToMarkdown(para, clipStartChars, clipEndChars)` — `code.js:2219`
- `extractTableCells(table)` — `code.js:2619-2629`:
```javascript
function extractTableCells(table) {
  var allCells = [];
  var rowCount = table.GetRowsCount();
  for (var r = 0; r < rowCount; r++) {
    var row = table.GetRow(r);
    for (var c = 0; c < row.GetCellsCount(); c++) {
      allCells.push({ r: r, c: c });
    }
  }
  return extractPartialTableCells(table, allCells);
}
```
- heading/list helpers (`getHeadingLevel`, `isListParagraph`, `getDrawingMarker`) — same sandbox.

The new extractor's leaf helpers MUST be defined INSIDE its own `callCommand` body (either factor the init body into a shared named fn with a `mode` branch reading `Asc.scope`, or duplicate the helpers). **Analog C — current selection-scoped entry + the SILENT fallback to make verbose (D-03).** Selection path entry (`code.js:2631-2641`):

## Shared Patterns

### ES5-strict plugin code
**Source:** project rule (MEMORY.md) + entire `code.js`.
**Apply to:** every excerpt added to `plugins/onlyoffice-scribe/scripts/code.js` and INSIDE any `callCommand` body.
No arrow functions, no `const`/`let` — `var` + `function` only.

### Live-ref read at the seam (no stale closure)
**Source:** `ScribeContext.jsx:86-89` (mirror pattern), read at `:166-169`.
**Apply to:** the new `includeDocumentRef` and `extractFullDocumentRef` — keep `sendMessage` deps `[client, t]` unchanged.

### Dedicated `reqId` bridge channel (bypasses the 1 MB cap)
**Source:** dev-test channel `code.js:3360-3376` (plugin) ↔ `View.jsx:121-132` listener idiom (host).
**Apply to:** all host↔plugin document round-trips. NEVER route the document over `cozy-bridge:intent` — `validateIntent`/`validateResponse` reject `JSON.stringify(data).length > MAX_DATA_SIZE` (`protocol.js:35,134,188`).
```javascript
// protocol.js:35
const MAX_DATA_SIZE = 1_000_000
// protocol.js:134 (validateIntent) — only applies to cozy-bridge:intent/:response
if (JSON.stringify(msg.data).length > MAX_DATA_SIZE) { /* reject */ }
```

### Function-injection via provider setter
**Source:** `setPanelActions` (`ScribeContext.jsx:152-154`, `View.jsx:237-242`).
**Apply to:** injecting `extractFullDocument` from `View.jsx` into `ScribeContext`.

### Frozen v3.1 contract (NON-negotiable, D-07)
**Source:** `scribeAI.js` (`RESPONSE_CONTRACT_CORE`/`CARDINALITY_CHAT`/`CHAT_PERSONA`/`markerPreservationClauses`), `scribeResponse.js`, `callScribeAIWithReask`.
**Apply to:** all prompt/response changes — append-only framing, no edits to frozen blocks. Re-validate with PROBE-01 (`scribeProbe.spec.js`) + corpus (`scribeResponse.corpus.spec.js`).
