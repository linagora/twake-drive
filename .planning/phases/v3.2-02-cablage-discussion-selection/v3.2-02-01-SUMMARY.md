# Phase v3.2-02 Plan 01: Câblage discussion + sélection Summary

Wired the « discussion » and « sélection » include checkboxes into the LLM prompt at the single `ScribeContext.sendMessage` composition seam — gating prior-turn history and the current selection deterministically — while keeping the hard-gated v3.1 `discussion`/`fragments` contract, single corrective re-ask, and contextual fallback byte-for-byte intact.

## What Was Built

- **Selection gate (current turn only, D-02/D-04):** `selectionIncluded = includeSelectionRef.current && !!selectionContext`. When off, the current `userMessage.selection` is `null` (D-03 faithful registry), the current user message carries no `[Selected text from document]…` block, and `buildChatSystemPrompt` receives `''` (no marker-preservation clauses). When on, behavior is exactly v3.1.
- **Discussion gate (D-01):** off ⇒ `aiMessages` = `[system, current user turn]` only; on ⇒ full prior history mapped exactly as v3.1 (prior user turns replay their stored `.selection` block as-is; prior assistant turns via `serializeAssistantTurnForHistory`). The gate only decides how many turns are mapped, never how a past turn renders (D-02).
- **Live-read (D-03, no stale closure):** `includeDiscussionRef` / `includeSelectionRef` mirror the booleans every render the same way `messagesRef.current` does. `sendMessage` reads `*.current`; its deps stay `[client, t]` so the callback identity is preserved (asserted in the compose spec).
- **D-05 framing:** `buildChatSystemPrompt` gained an optional 2nd `{ includeSelection, includeDiscussion }` flags arg. A new private `contextSourceFraming()` appends ~1 short English sentence per included source AFTER the contiguous `RESPONSE_CONTRACT_CORE + CARDINALITY_CHAT + marker` block. With no flags the output is byte-identical to before (regression safety).
- **Executable proof:** `ScribeContext.compose.spec.jsx` mocks `callScribeAIWithReask` to capture the assembled `aiMessages` with no network, asserting all four gate quadrants, the D-02 past-selection replay, the D-03 determinism + live-read, and the CTX-LLM-05 no-fragment-leak guard. `scribeAI.spec.js` extended with the D-05 framing cases.
