| Option | Description | Selected |

**User's choice:** Live — reformulé par l'utilisateur en « on ne stocke que ce qui est réellement envoyé ».

L'option snapshot est explicitement rejetée (et pour « discussion » contre-intuitive). → D-03.

## Architecture de prompt (multi-tours vs prompt structuré)

| Option | Description | Selected |
|--------|-------------|----------|
| α — Gater la structure v3.1 | Chat natif + bloc `[Selected text]` inline ; v3.2-02 ne fait que gater. Faible risque. | |
| β — Prompt structuré explicite maintenant | Cadre nommant chaque source + « juge la pertinence » dès maintenant. Plus pérenne mais redesign large touchant le contrat v3.1. | |
| γ — α maintenant, conçu pour β | Gater v3.1 maintenant, poser le point de composition pour basculer proprement en v3.2-03. | ✓ |

**User's choice:** γ + une touche de framing additive (option β légère) dans le system prompt.
**Notes:** L'utilisateur a soulevé la question de présenter explicitement les sources au LLM (« 1. historique 2. sélection 3. document, juge la pertinence »). Clarifications apportées : (1) « multi-tours » = tableau de messages à rôles, à conserver ; (2) avis technique — hybride (multi-tours pour la conversation + blocs nommés pour les ressources), ne pas aplatir en prompt unique, framing léger pour ne pas casser le contrat JSON sur Mistral-Small. Décision : gater maintenant (γ) + petite annonce conditionnelle des sources dans le system prompt (D-05) ; cadre énuméré complet + document reportés à v3.2-03. → D-00 / D-05.
