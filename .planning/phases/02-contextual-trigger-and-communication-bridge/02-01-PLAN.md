  <action>
Create the cozy-bridge standalone module in `src/lib/cozy-bridge/` with three files:

**types.js** -- JSDoc type definitions for the intent protocol:
- `@typedef IntentMessage` with fields: `type` (string, always `'cozy-bridge:intent'`), `version` (number, always 1), `intentId` (string, UUID), `action` (string, e.g. `'AI_TEXT_EDIT'`), `source` (string, sender identity), `data` (object, intent payload)
- `@typedef ResponseMessage` with fields: `type` (string, always `'cozy-bridge:response'`), `version` (number, always 1), `intentId` (string, correlation ID), `status` (string, `'ok'` or `'error'`), `action` (string, `'replace'`/`'insert'`/`'cancel'`), `data` (object, response payload)

**protocol.js** -- Constants and validators:
- Export constants: `PROTOCOL_VERSION = 1`, `MSG_TYPE_INTENT = 'cozy-bridge:intent'`, `MSG_TYPE_RESPONSE = 'cozy-bridge:response'`
- Export `createIntentMessage(action, data, source)` -- factory that generates a message with `crypto.randomUUID()` for intentId
- Export `createResponseMessage(intentId, status, action, data)` -- factory for response messages
- Export `validateIntent(msg)` -- returns boolean: checks msg is object, has correct type/version, has intentId (string), action (string), source (string), data (object). Log specific validation failure reasons to console.warn.
- Export `validateResponse(msg)` -- returns boolean: checks msg is object, has correct type/version, has intentId, status is 'ok' or 'error'
- Size limit validation: reject messages where `JSON.stringify(msg.data).length > 1_000_000` (1MB limit per user decision on schema validation)

**index.js** -- CozyBridge host-side class:
- Constructor takes `allowedOrigins` (array of strings)
- `onIntent(action, handler)` -- registers a handler function for an intent action. Handler receives `(intentMessage, respondFn)` where `respondFn({ status, action, data })` sends response back via `event.source.postMessage()`
- `_onMessage(event)` -- the raw message listener: (1) check `event.origin` against allowedOrigins, if not matched: `console.warn('[cozy-bridge] Ignored message from unknown origin:', event.origin)` and return, (2) check msg.type === MSG_TYPE_INTENT and msg.version === PROTOCOL_VERSION, (3) validate with `validateIntent()`, (4) look up handler via `this.handlers.get(msg.action)`, (5) call handler with msg and a respond function that uses `createResponseMessage()` and `event.source.postMessage(response, event.origin)`
- `destroy()` -- removes the message event listener (cleanup)
- Store `event.source` and `event.origin` when an intent is received, for response routing
- If `event.source` is null when responding, log error: `'[cozy-bridge] Cannot respond: event.source is null'`

All files use standard ES module syntax (import/export). No TypeScript, no JSX. Plain JavaScript with JSDoc type annotations.

**New: castIntent function (ES5 Promise-based):**
```javascript
function castIntent(action, data) {
  var intentId = generateIntentId();
  var message = {
    type: "cozy-bridge:intent",
    version: 1,
    intentId: intentId,
    action: action,
    source: "onlyoffice-plugin",
    data: data
  };
  window.top.postMessage(message, cozyOrigin);
  log("Intent cast: " + action + " id=" + intentId);
  // Store for correlation
  pendingIntents[intentId] = { action: action };
}
```
Note: Do NOT use Promise constructor (ES5 sandbox may not support it). Use a simple callback/pending-map pattern instead. The response comes via the message listener.

**New: generateIntentId function:**
- Try `crypto.randomUUID()` first, fall back to a simple timestamp+random pattern: `"intent-" + Date.now() + "-" + Math.random().toString(36).substr(2, 9)`

**New: Response message listener:**
```javascript
window.addEventListener("message", function(event) {
  var msg = event.data;
  if (!msg || msg.type !== "cozy-bridge:response" || msg.version !== 1) return;
  var pending = pendingIntents[msg.intentId];
  if (!pending) return;
  delete pendingIntents[msg.intentId];
  log("Intent response: action=" + msg.action + " status=" + msg.status);
  if (msg.status === "ok") {
    handleIntentResponse(msg);
  }
});
```

**New: handleIntentResponse function:**

- If `msg.action === "insert"`: use the InsertContent pattern from Phase 1 (Asc.scope + callCommand) to insert `msg.data.text` after the current selection, preserving original text
- If `msg.action === "cancel"`: do nothing (log only)
