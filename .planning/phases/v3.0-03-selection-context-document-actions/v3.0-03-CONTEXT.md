# Phase v3.0-03: Selection Context + Document Actions - Context

**Gathered:** 2026-03-18
**Status:** Ready for planning

<domain>
## Phase Boundary

User can reference selected document text in the chat panel and apply AI responses back to the document. Delivers: selection chip in chat input, selection included as context in AI prompts, Copy/Replace/Insert action buttons on AI messages. Does NOT include: panel resize (v3.0-04), streaming (future), persistence (future).

</domain>

<decisions>
## Implementation Decisions

### Selection chip
- Chip displayed in a dedicated zone ABOVE the textarea (not inline, not under header)
- Updates in real-time as the user selects text in OO (plugin already polls GetSelectedText every 500ms)
- Truncated to 1 line with ellipsis by default, expandable on click to show full text
- Dismiss button (×) to exclude the selection from the message — chip reappears if text is still selected in OO after next poll

### Action buttons on AI messages
- Copy and Insert always active (Insert inserts at cursor position even without selection)
- Replace only active when text is selected in OO; disabled/hidden otherwise
- Buttons always visible below the message content (not on hover) — focus set on a button for keyboard accessibility
- Copy copies as rich HTML to clipboard (not plain text, not markdown source)
- After successful action (Replace/Insert/Copy): button briefly changes state to ✔ confirmation, then reverts

### Selection → AI prompt integration
- Selection included as a user message composite: selection text embedded in the user's message (not in system prompt)
- This ensures multi-turn coherence: each message carries its own selection context, the AI sees which text was active for which question
- Selection text converted to Markdown from OO HTML (via Turndown — existing v2.1 pipeline)
- In the chat history display: user message shows selection as a visually separated block (quote/citation style) above the user's typed text

### Claude's Discretion
- Exact chip component styling (colors, border, icon)
- How to format the composite message for the AI (delimiter style, instructions)
- How to propagate selection state from useCozyBridge to ScribeContext
- Insert-at-cursor mechanism when no text is selected (plugin-side behavior)
- Keyboard focus strategy for action buttons

</decisions>

<specifics>
## Specific Ideas

- The selection chip should feel like a "context attachment" — similar to how ChatGPT shows attached files
- The quote block in user messages should be visually distinct but not overwhelming (subtle border-left or background)
- Action buttons should be small/compact — icons preferred over text labels to save space
- The ✔ confirmation state on buttons should be brief (~1.5s) and use a success color (green)

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- `useCozyBridge.js`: Already provides `pendingIntent.data.text` and `pendingIntent.data.html` — selection data is available
- `ScribeContext.jsx`: Has `addMessage()` — extend message schema to carry selection metadata
- `scribeConversion.js`: Has `markdownToHtml()` via marked — and Turndown is available for HTML→markdown
- `ScribeResultPanel.jsx`: Has working Replace/Insert/Cancel buttons — pattern to follow for chat action buttons
- `View.jsx`: Has `handleReplace(text)` and `handleInsert(text)` with `respond()` protocol — reuse for panel actions

### Established Patterns
- Intent-based communication: Plugin → postToAncestors → useCozyBridge → React state
- Respond protocol: `respond({ status, action, data: { text, html } })` posts back to plugin for document modification
- Message shape: `{ id, role, content, timestamp }` — needs extension for selection context
- PasteHtml with smart spacing for rich content injection (single undo point)

### Integration Points
- `ChatInput.jsx`: Add selection chip zone above textarea
- `ChatMessageList.jsx`: Add action buttons below AssistantBubble
- `ScribeContext.jsx`: Expose current selection state + extend message schema
- `View.jsx`: Route respond() to panel actions (currently only wired to popover)
- Plugin `code.js`: May need a new intent for "insert at cursor" without active selection

</code_context>
