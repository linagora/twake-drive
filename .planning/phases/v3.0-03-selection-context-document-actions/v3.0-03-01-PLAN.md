**ScribeContext.jsx changes:**
- Add `currentSelection` state: `{ text, html, markdown }` or `null`
- Add `setCurrentSelection(text, html)` callback that converts html to markdown via `htmlToMarkdown` from scribeConversion.js and stores `{ text, html, markdown }`
- Add `dismissSelection()` callback that sets a `selectionDismissed` ref to true (prevents chip from reappearing until a NEW different selection arrives)
- Expose `currentSelection`, `setCurrentSelection`, `dismissSelection` in provider value
- When `setCurrentSelection` is called with a text that differs from the currently dismissed text, reset `selectionDismissed` to false

**View.jsx changes:**
- Extract `setCurrentSelection` from `useScribe()` (null-safe like other scribe values)
- In a useEffect, whenever `showScribeButton` changes and has text, call `setCurrentSelection(showScribeButton.text, null)` (showScribeButton only has text, no html -- pass null for html)
- Also when `pendingIntent` arrives with data, call `setCurrentSelection(pendingIntent.data.text, pendingIntent.data.html)` -- this gives richer context when available
- When `showScribeButton` becomes null (deselection), call `setCurrentSelection(null, null)` to clear selection

**SelectionChip.jsx (new file):**
- Receives `selection` (the currentSelection object) and `onDismiss` callback as props
- Renders a compact chip-like element: left border accent (SCRIBE_PURPLE), truncated text (1 line, ellipsis), dismiss x button
- Click on chip body toggles expanded state (shows full text, wraps)
- Uses useTheme() for dark/light theming (background: theme.palette.action.hover)
- Dismiss button calls onDismiss
- Inline styles (consistent with project pattern -- no stylus for Scribe components)
- Max display text: use `selection.text` for display (plain text), truncated via CSS `overflow: hidden; text-overflow: ellipsis; white-space: nowrap` when collapsed

**ChatInput.jsx changes:**
- Import SelectionChip and useScribe
- Read `currentSelection` and `dismissSelection` from useScribe()
- Render SelectionChip ABOVE the existing textarea+button row, inside the same outer div but before the textarea row
- Only render SelectionChip when `currentSelection` is not null
- The borderTop stays on the outer div; the chip sits between border and textarea

**Locale files:**
- en.json: Add `Scribe.chat.selection_context` = "Selected text" (for accessibility/screen readers)
- fr.json: Add `Scribe.chat.selection_context` = "Texte selectionne"

**ScribeContext.jsx -- extend sendMessage:**
- Accept optional second parameter: `sendMessage(text, selectionContext)` where selectionContext is `{ text, markdown }` or null
- When selectionContext is provided, build a composite content string for the AI:
  ```
  [Selected text from document]
  {selectionContext.markdown}
  [End of selected text]

  {text}
  ```
- Store the message with extended shape: `{ id, role: 'user', content: text, selection: selectionContext, timestamp }` -- content is the user's typed text only (for display), selection is metadata
- When building aiMessages for the API call, use the composite format for messages that have selection (concatenate selection + user text)
- Messages without selection just use content as before

**ChatInput.jsx -- pass selection on send:**
- When handleSend fires and currentSelection is not null, call `sendMessage(trimmed, { text: currentSelection.text, markdown: currentSelection.markdown })` instead of `sendMessage(trimmed)`
- After sending, do NOT clear the selection -- it stays visible for follow-up questions about the same selection

**ChatMessageList.jsx -- render selection quote block:**

  - Italic text, fontSize 12, max 3 lines with overflow hidden (CSS line-clamp)
  - Margin-bottom 6px between quote and user text
- In the messages.map(), pass `msg.selection` to UserBubble when it exists
