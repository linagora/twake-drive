## Accomplishments
- ScribeContext exposes currentSelection state with set/dismiss callbacks and selectionDismissedRef pattern
- SelectionChip renders truncated selection above textarea with expand/collapse and dismiss
- View.jsx feeds showScribeButton and pendingIntent selection data into ScribeContext
- AI receives composite prompt with delimited selection markdown when user sends with active selection
- Chat history shows purple-bordered quote block above user text for selection-attached messages

## Decisions Made
- selectionDismissedRef tracks dismissed text to prevent chip re-showing until new different selection arrives
- Selection persists after sending message for follow-up questions about same selection
- Composite AI prompt uses `[Selected text from document]` / `[End of selected text]` delimiters for clear context separation
- htmlToMarkdown conversion used for rich selection; plain text fallback when no HTML available
