### Hook Ordering Note

`View.jsx` lines 100-104 contain a `useEffect` that references `handleReplace` and `handleInsert` in its body and dependency array, but these `useCallback` definitions appear later at lines 132 and 141. This is syntactically unusual but functionally correct in React: all hooks in a render pass are evaluated before any effect bodies fire, so `handleReplace` and `handleInsert` are fully initialized by the time the effect runs. React's rules of hooks do not require that hook declarations precede their use in the same component body — only that hooks are called unconditionally and in the same order each render. ESLint may flag this as a stylistic concern, but it does not introduce a runtime bug.
