}, [respond, focusEditor])

**ScribeContext.jsx changes:**
- Add `panelActions` state (object or null): `{ replace: fn, insert: fn }`
- Add `setPanelActions(actions)` callback to store action handlers from View.jsx
- Expose `panelActions`, `setPanelActions`, and `currentSelection` in provider value (currentSelection already from Plan 01)

**View.jsx changes:**
- Extract `setPanelActions` from useScribe() (null-safe)
- In a useEffect, call `setPanelActions({ replace: handleReplace, insert: handleInsert })` when handlers are stable
- This passes the respond()-based handlers into the ScribeContext so MessageActions can call them
- Cleanup: set panelActions to null on unmount

**MessageActions.jsx (new file):**
- Receives props: `content` (message markdown text), `hasSelection` (boolean)
- Renders a row of compact icon buttons below the message: Copy, Insert, Replace (conditionally)
- Uses inline styles (project pattern), theme-aware via useTheme()
- **Copy button (always visible):**
  - Icon: clipboard/copy SVG icon (inline)
  - On click: convert `content` (markdown) to HTML via `markdownToHtml`, then use `navigator.clipboard.write()` with `ClipboardItem` containing `text/html` blob AND `text/plain` blob (plain text fallback = content as-is). Per user decision: copies as rich HTML, not plain text.
  - Fallback for browsers without ClipboardItem: use `navigator.clipboard.writeText(content)` (plain text only)
- **Insert button (always visible):**
  - Icon: insert/add SVG icon (inline)
  - On click: call `panelActions.insert(content)` from useScribe()
  - Works with or without selection (inserts at cursor per existing plugin behavior)
- **Replace button (only when hasSelection is true):**
  - Icon: swap/replace SVG icon (inline)
  - On click: call `panelActions.replace(content)` from useScribe()
  - When hasSelection is false, do not render this button at all
- **Confirmation state:**
  - Each button has local state: `confirmedAction` (null, 'copy', 'insert', 'replace')
  - On successful action, set confirmedAction to the action name, show checkmark icon with green color (#22c55e)
  - After 1500ms timeout, reset confirmedAction to null
  - Only one confirmation at a time (setting a new one clears the old)
- Button styling: small (24x24), borderRadius 4, transparent background, hover: theme.palette.action.hover, gap 4 between buttons, margin-top 4
- Keyboard: buttons are focusable, Enter/Space triggers action

- After each AssistantBubble in the messages.map(), render `<MessageActions content={msg.content} hasSelection={!!currentSelection} />` as a sibling below the bubble
