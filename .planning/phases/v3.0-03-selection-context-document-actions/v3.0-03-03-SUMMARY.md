# v3.0-03-03 — MessageActions Replace/Insert functional + enriched selection

## Problem

Two issues discovered during panel testing:

1. **Replace/Insert no-op**: clicking Replace or Insert on a panel AI message did nothing. Root cause: `respond()` in useCozyBridge early-returns silently when no pending intent exists (respondRef is null). In a pure chat flow (user types in panel, no inline popover triggered), no AI_TEXT_ASSISTANT intent was ever emitted by the plugin, so respondRef was always null.

2. **Selection chip lost formatting**: the panel's SelectionChip showed unstyled text. Root cause: `SELECTION_CHANGED` intent carried only `{text, html}`, while `AI_TEXT_ASSISTANT` carried `{text, enrichedMd, tableAmbiguity, partialTableInfo}` via `buildEditIntentData()`. The panel fell back to Turndown (lossy HTML→MD) instead of using the enriched markdown.

## Solution

### PANEL_ACTION intent (Tasks 1-3)

New one-way cozy-bridge intent type `PANEL_ACTION` that bypasses the pending-intent/respond mechanism:

- `protocol.js`: added `PANEL_ACTION` constant + `createPanelActionIntent({action, text, html, md})` factory
- `useCozyBridge.js`: added `castPanelAction` callback that broadcasts the intent down to all descendant frames (recursive walk pattern)
- `View.jsx`: `handleReplace`/`handleInsert` now route to `castPanelAction` when no pendingIntent exists, `respond()` when one exists. Uses `pendingIntentRef` to avoid handler churn on every selection change.
- `code.js` (ES5): new message listener for `cozy-bridge:intent` + `PANEL_ACTION`. Synthesizes a response-shaped object and calls the existing `handleIntentResponse` function — zero behavioral drift between inline and panel injection paths.

### Enriched SELECTION_CHANGED (Task 4)

- `code.js`: replaced inline `{text, html}` payload in SELECTION_CHANGED with `buildEditIntentData()` output + html field. Now carries `enrichedMd`, `tableAmbiguity`, `partialTableInfo`.
- `View.jsx`: `handleSelectionChanged` passes `data.enrichedMd` to `setCurrentSelection` and stores `data.partialTableInfo` in `partialTableInfoRef`.
- `ScribeContext.jsx`: `setCurrentSelection` accepts optional `enrichedMd` third parameter. Uses it as `markdown` when present, falls back to `htmlToMarkdown(html)` when not.

## Known Limitations

- **Table round-trip via panel**: tables are mapped with `[TABLE:N]` markers but the current injection relies on volatile in-memory `ApiTable.Copy()` clones. In the panel flow, the original table may be modified between extraction and injection. A portable table snapshot approach is planned in v3.0-03-05.
- **partialTableInfo in pure chat flow**: if no inline popover was opened earlier in the session, `partialTableInfo` will be undefined for panel actions, so partial-table handling won't kick in. This matches expected behavior.
- **Focus after panel action**: `setTimeout(focusEditor, 100)` runs after panel Replace/Insert, moving focus to the editor. Acceptable since user likely wants to see the result.
