<objective>
Replace volatile in-memory table clones (ApiTable.Copy()) with portable table snapshots captured at extraction time, so table formatting survives the full round-trip regardless of document changes.

Purpose: The current table pipeline relies on finding and cloning the original table in the document during injection. This is fragile: in the panel flow, the user may modify or delete the table between extraction and injection (multi-turn conversation, minutes of delay). Even in the inline flow, the clone only lives within a single callCommand scope. A portable snapshot that travels with the enrichedMd makes the pipeline robust for both modes.

Output: At extraction, each table's full structure and formatting is captured as a serializable snapshot. At injection, the table is reconstructed from this snapshot (not from the document), with modified cell content from the LLM applied on top.
</objective>

**Extraction** (selection scan, inside callCommand):
- Tables detected via `doc.GetAllTables()` + range overlap checks (code.js:2514-2522)
- Cell content extracted as markdown via `extractTableCells()` / `extractPartialTableCells()` (code.js:2464-2487)

- Table document indices stored in `tableDocIndices[]` for later lookup
- Output: `enrichedMd` with `[TABLE:N][CELL:r,c]...[/CELL]...[/TABLE]` markers

**Injection** (inside callCommand for Replace/Insert):
- `tableClones = {}` created at line 779, populated at lines 873-998

- `ApiTable.Copy()` creates a volatile in-memory clone (lines 874, 963, 987)

- Clone pushed to `content[]` array for `InsertContent` (line 1250)
- Font preserved per cell via `cFonts[r,c]` captured from original table (lines 918-947)

**Fragility points:**
1. `tableDocIndices` assumes table positions haven't changed — any insertion/deletion above shifts indices
2. `ApiTable.Copy()` clone lives only within the callCommand scope — not serializable, not storable
3. No formatting metadata travels with the enrichedMd — all formatting comes from the (possibly stale) original

<proposed_architecture>

## New Architecture: Portable Table Snapshots

```
EXTRACTION (callCommand)                    TRANSPORT                        INJECTION (callCommand)
────────────────────────                    ─────────                        ──────────────────────
For each TABLE:N in selection:              enrichedMd +                     For each [TABLE:N] in LLM response:
                                            tableSnapshots[]
│                                           travels via:                     │
├─ [TABLE:N][CELL:r,c] markers             - SELECTION_CHANGED              ├─ Parse modified cells from LLM md
│  (text content → LLM)                    - AI_TEXT_ASSISTANT               │
│                                           - ScribeContext state             ├─ Rebuild table from snapshot:
└─ tableSnapshots[N] = {                   - PANEL_ACTION payload            │  ├─ Api.CreateTable(rows, cols)
     rows: 3,                                                                │  ├─ Apply cell formats from snapshot
     cols: 4,                                                                │  │  (borders, bg, font, merges)
     cells: [                                                                │  ├─ Fill cells with LLM-modified text
       { r:0, c:0,                                                           │  │  via replaceCellContent()
         content: [{                                                         │  └─ Apply table-level properties
           text: "Header",                                                   │     (width, alignment)
           bold: true,                                                       │
           italic: false,                                                    └─ Push table to content[] for InsertContent
           fontFamily: "Arial",
           fontSize: 11,
           underline: false,
           strikethrough: false,
           color: {r:0,g:0,b:0}
         }],
         bgColor: {r:230,g:230,b:230},
         borders: { top: {...}, ... },
         hMerge: 1, vMerge: 1
       },
       ...
     ],
     tableWidth: 9026,
     tableAlignment: "left"
   }
```

**Key principle**: the snapshot is JSON-serializable and self-contained. It contains everything needed to reconstruct the table with full formatting, independent of the original document state.

</proposed_architecture>

<format_decision>
## Why structured JSON (not HTML)

| Approach | Pros | Cons |
|----------|------|------|
| **HTML snapshot via GetSelectedContent** | Simple extraction | Hard to modify individual cells before injection; PasteHtml loses some OO properties; can't selectively update one cell |
| **Structured JSON snapshot** (chosen) | Full control; easy cell-level modification; Builder API can reconstruct precisely; JSON-serializable | More extraction code; must enumerate all relevant properties |
| **Full Copy() serialization** | Perfect fidelity | Not JSON-serializable; OO internal format, undocumented |

The structured JSON approach wins because:
1. We need to **modify individual cells** (LLM changes content) before injection — HTML makes this fragile (regex on HTML table cells)
2. Builder API (`Api.CreateTable`, `SetBold`, `SetFontFamily`, etc.) gives precise control over reconstruction
3. JSON travels cleanly through postMessage, ScribeContext state, and PANEL_ACTION payloads
4. The extraction code can progressively add properties (start with font/bold/italic, add borders/bg later)
</format_decision>

  <name>Task 4: Rewrite table injection to reconstruct from snapshots instead of document clones</name>
  <files>plugins/onlyoffice-scribe/scripts/code.js</files>

**ES5 syntax only. This is the critical task.**

Replace the current injection logic (which finds the original table by document index and calls Copy()) with reconstruction from the portable snapshot.

**New function: `rebuildTableFromSnapshot(snapshot, parsedCells)`**

Given a table snapshot (from Task 2) and parsed cell content from the LLM response:

1. Create a new table: `Api.CreateTable(snapshot.cols, snapshot.rows)`
2. Apply table-level properties: width, alignment, default borders (based on Task 1 research)
3. For each cell in the snapshot:
   a. Get the cell: `table.GetRow(r).GetCell(c)`
   b. Apply cell formatting: bgColor, borders, width, merges (from snapshot)
   c. If the LLM modified this cell (matching `[CELL:r,c]` in parsed output):
      - Apply the LLM content using the existing `replaceCellContent()` function
      - Use font from snapshot (not from original document)
   d. If the LLM did NOT modify this cell:
      - Rebuild the original content from `snapshot.cells[i].content` (array of runs with formatting)
4. Return the table object for InsertContent

**Integration:**
- Replace the current `origTable.Copy()` calls (lines 874, 963, 987) with `rebuildTableFromSnapshot(snapshot, parsedCells)`
- Remove `tableDocIndices` lookup — no longer needed
- `cFonts` (per-cell font capture from original) replaced by snapshot font data
- Partial table handling: if `selectedCells` is a subset, only those cells should be in the snapshot. Rebuild only modifies those cells; unselected cells get their original content from the snapshot.

**Backward compatibility:**
- If `tableSnapshots` is not present (e.g. older cached selection data), fall back to the existing Clone-from-document path. This ensures graceful degradation during the transition.

**Remove volatile code** (once snapshot path is verified):
- Remove `tableDocIndices` storage in Asc.scope
- Remove `doc.GetAllTables()[index]` lookups
- Keep `ApiTable.Copy()` only in the fallback path (or remove entirely if confident)
