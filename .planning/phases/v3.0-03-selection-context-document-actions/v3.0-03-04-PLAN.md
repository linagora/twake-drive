<objective>
Replace volatile in-memory table clones (ApiTable.Copy()) with portable ToJSON snapshots captured at extraction time, so table formatting survives the full round-trip regardless of document changes.

Purpose: The current table pipeline relies on finding and cloning the original table in the document during injection. This is fragile: in the panel flow, the user may modify or delete the table between extraction and injection (multi-turn conversation, minutes of delay). Even in the inline flow, the clone only lives within a single callCommand scope. A portable snapshot that travels with the enrichedMd makes the pipeline robust for both modes.

Output: At extraction, each table is serialized via `ApiTable.ToJSON(true, true)` (lossless, captures all formatting + images). At injection, the table is reconstructed via `Api.FromJSON(snapshot)` which returns an ApiTable directly, then modified cells are updated with the LLM's content via the existing `replaceCellContent()` function.
</objective>

<proposed_architecture>

## New Architecture: ToJSON / FromJSON Table Snapshots

```
EXTRACTION (callCommand)                    TRANSPORT                        INJECTION (callCommand)
────────────────────────                    ─────────                        ──────────────────────
For each TABLE:N in selection:              enrichedMd +                     For each [TABLE:N] in LLM response:
                                            tableSnapshots[]
│                                           travels via:                     │
├─ [TABLE:N][CELL:r,c] markers             - SELECTION_CHANGED              1. table = Api.FromJSON(snapshot)
│  (text content → LLM)                    - AI_TEXT_ASSISTANT                 → ApiTable with ALL original
│                                           - ScribeContext state               formatting (lossless)
└─ tableSnapshots[N] =                     - PANEL_ACTION payload            │
     table.ToJSON(true, true)                                               2. replaceCellContent() on cells
     (JSON string, lossless)                                                   that the LLM modified
                                                                               (existing code, unchanged)
     Captures EVERYTHING:                                                    │
     - borders, colors, fonts                                               3. Push table to content[]
     - images                                                                  for InsertContent
     - merges, widths
     - styles, numberings
     - any OO internal property
```

**Key advantages:**
1. **Lossless** — ToJSON(true, true) captures everything OO knows about the table (borders, backgrounds, merges, fonts, images, styles, numberings)
2. **Synchronous** — both ToJSON and FromJSON work inside callCommand (unlike PasteHtml which is async)
3. **Direct ApiTable return** — FromJSON returns an ApiTable directly, no need to search for a pasted table
4. **JSON-serializable** — the snapshot is a string, travels via postMessage like any other field
5. **Single callCommand** — extraction and injection each happen in one callCommand, one undo point
6. **No manual property enumeration** — no need to read/write individual borders, bg, merges, fonts
7. **Existing cell modification code reused** — replaceCellContent() works on the FromJSON table exactly like on a Copy() clone

**Research findings** (from Task 1, see table-snapshot-research.md):
- `ApiTable.ToJSON(true, true)` — the two `true` flags enable writing styles and numberings, ensuring images and complex formatting are preserved
- `Api.FromJSON(jsonString)` — returns the reconstructed ApiTable synchronously
- Both tested in callCommand context via sdkjs source analysis
- PasteHtml was ruled out (async, can't run in callCommand)
- CopyProcessor HTML extraction exists but is less reliable than ToJSON for round-trip

</proposed_architecture>

  <name>Task 4: Rewrite table injection to use FromJSON + cell modification instead of Clone</name>
  <files>plugins/onlyoffice-scribe/scripts/code.js</files>
  <action>
**ES5 syntax only. This is the critical task.**

Replace the current injection logic with the ToJSON/FromJSON approach.

**New injection flow for each [TABLE:N] in the LLM response:**

Inside the injection callCommand (where `tableClones` is currently built):

1. **Retrieve snapshot**: get `tableSnapshots[N]` from `Asc.scope.tableSnapshots` (set by the host's respond/castPanelAction payload, stored before callCommand via Asc.scope).

2. **Reconstruct table**: `var table = Api.FromJSON(tableSnapshots[N]);`
   - This returns a fully-formed ApiTable with all original formatting, images, content
   - Synchronous, runs inside callCommand

3. **Modify cells with LLM content**: for each cell that the LLM modified (matched from parsed `[CELL:r,c]` output):
   - Get the cell: `table.GetRow(r).GetCell(c)`
   - Call the existing `replaceCellContent(cellContent, parsedCell, fontFamily, fontSize)`
   - For `fontFamily` and `fontSize`: read from the reconstructed table's cell (it has the correct original fonts from the snapshot)
   - Cells NOT modified by the LLM are left untouched (they already have original content from FromJSON)

4. **Push to content[]**: `content.push(table)` — same as current `content.push(tableClones[ptIndex])`

**What this replaces:**
- Remove `origTable = doc.GetAllTables()[tableDocIndices[i]]` lookups — no longer needed
- Replace `origTable.Copy()` calls (lines ~874, 963, 987) with `Api.FromJSON(snapshot)`
- Remove `cFonts` per-cell font capture loop (lines ~918-947) — read font from the FromJSON table's cells directly
- Simplify partial table handling: FromJSON reconstructs the full table, modify only selected cells, done

**Backward compatibility:**
- If `tableSnapshots` is not present or `tableSnapshots[N]` is null/undefined (e.g. older cached selection data, or ToJSON failed during extraction):
  ```js
  if (tableSnapshots && tableSnapshots[N]) {
    table = Api.FromJSON(tableSnapshots[N]);
  } else {
    // Fallback to existing Clone-from-document path
    var origTable = doc.GetAllTables()[tableDocIndices[ptIndex]];
    table = origTable.Copy();
  }
  ```
- This ensures graceful degradation during the transition and if ToJSON ever fails.

**Error handling:**
- Wrap `Api.FromJSON` in try/catch. If it throws, fall back to Clone path.
- Log: `log('[Scribe] FromJSON failed for TABLE:' + N + ', falling back to clone');`

**Asc.scope for snapshots in injection callCommand:**
- The `tableSnapshots` array needs to be available inside the injection callCommand.
- Set `Asc.scope.tableSnapshots = receivedTableSnapshots` before calling callCommand, alongside existing Asc.scope fields (`parsedTables`, `parsedBlocks`, etc.)
- Read inside callCommand: `var tableSnapshots = Asc.scope.tableSnapshots;`
