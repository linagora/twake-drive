Create `tableCellMarkers.js` with four exported functions:

1. **`parseCellMarkers(text)`** — Parses `[CELL:r,c]text[/CELL]` markers from a string.
   - Regex: `/\[CELL:(\d+),(\d+)\]([\s\S]*?)\[\/CELL\]/g`
   - Returns array of `{ row: Number, col: Number, text: String }`.
   - Reset regex lastIndex before exec loop (or use a non-global regex with matchAll, or recreate regex inside function).
   - Returns empty array if no markers found.

2. **`validateCellCount(extractedMd, responseMd)`** — Compares cell counts between the extraction (enrichedMd) and the LLM response.
   - Calls `parseCellMarkers` on both strings.
   - Returns `{ valid: boolean, expected: number, actual: number }`.
   - Valid if actual >= expected (LLM may duplicate but should not drop cells).
   - If no cells in extractedMd, returns `{ valid: true, expected: 0, actual: 0 }` (no table content).

3. **`cellsToMarkdownTable(cells)`** — Converts parsed cells array to a GFM pipe-table string.
   - Determine grid dimensions from max row/col values.
   - Build a 2D grid array, fill from cells.
   - For each cell text: `.trim()` and replace newlines with space (GFM pipe tables cannot have multi-line cells).
   - Row 0 = header row, insert separator line (`| --- | --- |`) after it.
   - Return empty string if cells array is empty.

4. **`transformCellMarkersForPreview(md, enrichedMd)`** — Pre-processes an LLM response for MarkdownPreview display.
   - If `md` does not contain `[CELL:`, return `{ displayMd: md, warning: null }`.
   - Parse cells from `md` via `parseCellMarkers`.
   - Validate against `enrichedMd` via `validateCellCount`.
   - Build pipe-table via `cellsToMarkdownTable`.
   - Replace the entire cell marker block in `md` with the pipe-table string. Use regex to find the contiguous block of `[CELL:...]...[/CELL]` lines (possibly separated by newlines) and replace with the table string.
   - If validation fails, set `warning` to a string like `"Table cell count mismatch: expected {expected}, got {actual}"`.
   - Return `{ displayMd: transformedString, warning: string|null }`.

Add JSDoc on each function. Use ES module exports (not CommonJS).
