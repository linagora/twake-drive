| T-v3.1-02-03 | Information Disclosure | serializeAssistantTurnForHistory leaking full fragment bodies / stale fragments back into model context | mitigate | Serializer emits discussion + a truncated compact note only (TRUNCATE constant); never the full fragment body, never raw contract JSON (ARCHITECTURE §2 Option B, PITFALLS Pitfall 8). |
| T-v3.1-02-04 | Spoofing | Marker injection — fragment body or document text containing literal `{{fragment:0}}` mis-binding a card | mitigate | extractChannelMarkers scans only the channel passed by the caller; parseScribeResponse cross-checks markers against fragments.length and flags out-of-range as a non-fatal warning. Render-side honoring of markers only in discussion is a v3.1-04 concern (out of scope here). |
