# Phase v3.1-04 Plan 01: assistantSegments Placement Helper Summary

Pure, unit-tested `buildAssistantSegments(discussion, fragments)` that converts the `{discussion, fragments[]}` contract model into an ordered `{type:'prose'|'card'}` segment array, isolating all REF-safe fragment placement (CONTRACT-02 / FRAG-01 / D-04) from the React render so a `<FragmentCard>` node can be emitted per marker instead of a substituted string.

## What Was Built

- **`assistantSegments.js`** — exports `buildAssistantSegments`. Algorithm: coerce non-string discussion → `''` and non-array fragments → `[]`; with 0 fragments return a single `{type:'prose', md}` segment (no cards, CONTRACT-02/D-05); otherwise locate ordered `{{fragment:N}}` hits via `extractChannelMarkers(disc,'fragment')`, filter to in-range indices, forward-walk pushing prose then card segments, strip dangling out-of-range markers from the tail prose, and append orphan (unreferenced) fragments as cards at the end in fragment order (D-04). Each card's `raw` is `fragments[index]` unmodified (D-03).
- **`assistantSegments.spec.js`** — 11 pure Jest tests (no React/RTL), mirroring the `scribeResponse.spec.js` idiom: CONTRACT-02 (0 frags → 1 prose, no card; empty discussion case), FRAG-01 (inline card at marker, 0-indexed), D-04 (orphans at end; mixed inline + orphan), REF-safety/CONTRACT-04 (`{{REF:scribe-ref-2:Figure 1}}` survives verbatim in prose, only fragment marker consumed), out-of-range skip (`{{fragment:5}}` stripped, no card index 5), card.raw verbatim, and non-string/non-array guards.

## Decisions Made

- Forward-walk cursor emits segments in document order, replacing the old back-to-front substitution loop (which only needs to preserve string offsets, not emit an ordered list).
- The lone literal `/\{\{fragment:(\d+)\}\}/g` regex is restricted to the post-cursor tail strip (matching `composeAssistantDisplay`'s out-of-range behavior); all marker *location* goes through the REF-safe `extractChannelMarkers`, so cross-ref markers can never be consumed.

## Threat Surface

Threat register mitigations applied and test-guarded:
- T-v3.1-04-02 (crafted indices): `filter(index >= 0 && index < frags.length)`; out-of-range markers stripped from prose, never resolved to a card — covered by the out-of-range test.
- T-v3.1-04-03 (REF consumed by greedy regex): only `extractChannelMarkers('fragment')` used for location — covered by the REF-safety test.

No new security surface introduced (pure string slice/copy; no render, no eval, no network, no package installs).
