<objective>
Add full keyboard navigation across the chat thread using the repo's manual-index pattern (NO
roving-tabindex library — consistent with ScribeActionMenu/ScribeResultPanel). A thread-level focus
controller in `ChatMessageList` holds a flat, thread-ordered list of fragment-card button groups and a
`(cardPos, buttonPos)` focus coordinate. `ChatInput` gains an Up-from-empty-input hook and exposes an
imperative `focus()`. `ScribePanel` (the parent that renders both as siblings) wires the cross-component
callbacks: input Up → controller focuses the most-recent card's Insert button; controller Escape and
Down-past-newest → input.focus().

Keyboard contract:
- KBD-01: Up from empty input → most-recent card's Insert button; D-10 — if the most-recent message is
  pure discussion (0 cards), walk backwards to the last message that has cards; no wrap.
- KBD-02: Left/Right cycle Copy/Insert/Replace on the focused card (Replace present only when
  hasSelection — button group length 2 or 3).
- KBD-03: Up/Down move across all cards in thread order; Down past the most-recent card → input; no wrap
  upward past the oldest.
- KBD-04: Escape → input; Enter/Space activate the focused button (native button activation; controller
  only intercepts arrows with preventDefault).

Purpose: KBD-01..04 — complete keyboard parity with mouse, mirroring proven in-repo focus patterns.
Output: controller in `ChatMessageList.jsx`, hooks in `ChatInput.jsx`, wiring in `ScribePanel.jsx`,
plus `ChatMessageList.kbd.spec.jsx`.
</objective>
