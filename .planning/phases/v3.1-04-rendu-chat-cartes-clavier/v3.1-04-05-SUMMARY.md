# Phase v3.1-04 Plan 05: Keyboard navigation Summary

Full keyboard parity for the chat thread via a manual-index focus controller in `ChatMessageList` (no new dependency): `(cardPos, buttonPos)` coordinate over a flat, DOM-derived, thread-ordered list of fragment-card button groups, with `ChatInput` gaining an Up-from-empty hook + imperative `focus()` and `ScribePanel` wiring the two siblings.

## What was built

- **ChatInput** (`forwardRef`): `useImperativeHandle` exposes a single-shot `focus()`; `handleKeyDown` calls `onArrowUp()` only when the draft is empty (`text.length === 0`), so Up still edits a multi-line draft when text is present. Open-time focus burst untouched.
- **ChatMessageList** (`forwardRef`): a `keydown` handler on the scroll container intercepts arrows + Escape (`preventDefault`); Enter/Space pass through to native button activation.
  - KBD-02: Left/Right cycle (modulo) within the focused card's button group (length 2 or 3, gating-aware).
  - KBD-03: Up/Down move `cardPos` across the thread, preserving `buttonPos` clamped to the new group; Down past the newest card → `returnFocusToInput()`; Up past the oldest clamps (no wrap).
  - KBD-04: Escape → `returnFocusToInput()`; Enter/Space native.
  - KBD-01 + D-10: `focusMostRecentCardInsert()` (exposed via `useImperativeHandle`) focuses the last card's Insert button; because the on-demand list skips card-less messages, a pure-discussion newest message naturally resolves back to the last carded message. No-op when no cards exist anywhere.
- **FragmentCard**: root div tagged `data-fragment-card` so the controller collects cards in thread (DOM) order.
- **ScribePanel**: owns `inputRef` + `listRef`; passes `onArrowUp={() => listRef.current.focusMostRecentCardInsert()}` to ChatInput and `returnFocusToInput={() => inputRef.current.focus()}` to ChatMessageList. ScribeContext untouched.
