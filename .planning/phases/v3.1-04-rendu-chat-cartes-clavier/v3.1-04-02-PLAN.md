<objective>
Extend `MarkdownPreview`'s existing string-preprocess stage (which already converts `{{IMG:…}}`
markers) to cosmetically render the three remaining plugin markers per D-02:
`{{REF:scribe-ref-N:visible text}}` → visible text only, `[TABLE:N]`/`[CELL:r,c]` → a GFM pipe-table
(via the existing `transformCellMarkersForPreview` helper), and `[^scribe-fn-N]` → a discreet
superscript. The transform is COSMETIC ONLY: it produces a new string for display and never mutates
the caller's source (D-03). This is what makes fragment cards (Plan 03) and prose segments (Plan 04)
show clean rendered content while the raw markers stay intact for the insert/replace pipeline.

Purpose: D-02 — chat must show rendered tables/cross-refs/footnotes, not literal markers, without ever
losing the raw markers the OO reinjection pipeline needs.
Output: extended `MarkdownPreview.jsx` + `MarkdownPreview.spec.jsx`.
</objective>
