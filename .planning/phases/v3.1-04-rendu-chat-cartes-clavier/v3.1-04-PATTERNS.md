### `Scribe/FragmentCard.jsx` (NEW — component, request-response)

**Analogs:** `MessageActions.jsx` (the 3 handlers, verbatim) + `AssistantBubble` / `SelectionQuote`
(ChatMessageList.jsx:100-159) for the bordered-surface + Scribe-purple accent style.

**Surface/accent pattern** — reuse the exact Scribe-purple framing already used by `SelectionQuote`
(ChatMessageList.jsx:13, 98-119):

```jsx
//   borderRadius: 8, padding: '8px 10px'.  (radii/spacing are Claude's discretion, D-01.)
```

**Display vs raw split (D-03)** — pass the RAW fragment to BOTH the preview and the actions; the
preview cleans cosmetically *inside* `MarkdownPreview`, the actions never see the cleaned string:
```jsx
const FragmentCard = ({ raw, hasSelection }) => (
  <div style={{ border: `1px solid ${SCRIBE_PURPLE}`, background: SCRIBE_PURPLE_08,
                borderRadius: 8, padding: '8px 10px' }}>
    <MarkdownPreview>{raw}</MarkdownPreview>   {/* cosmetic cleanup happens INSIDE MarkdownPreview */}
    <MessageActions content={raw} hasSelection={hasSelection} />  {/* RAW → clipboard + panelActions */}
  </div>
)
```

**Action handlers to reuse verbatim** — `MessageActions` already takes a raw `content` string; do not
re-implement (MessageActions.jsx:56-86):
```jsx
const handleCopy = useCallback(async () => {
  const html = stripParagraphWrapper(markdownToHtml(content).trim())
  if (typeof ClipboardItem !== 'undefined') {
    const htmlBlob = new Blob([html], { type: 'text/html' })
    const textBlob = new Blob([content], { type: 'text/plain' })
    await navigator.clipboard.write([
      new ClipboardItem({ 'text/html': htmlBlob, 'text/plain': textBlob })
    ])
  } else { await navigator.clipboard.writeText(content) }
  showConfirmation('copy')
}, [content, showConfirmation])

const handleInsert = useCallback(() => {
  if (panelActions && panelActions.insert) { panelActions.insert(content); showConfirmation('insert') }
}, [content, panelActions, showConfirmation])

const handleReplace = useCallback(() => {
  if (panelActions && panelActions.replace) { panelActions.replace(content); showConfirmation('replace') }
}, [content, panelActions, showConfirmation])
```

**Replace gating (FRAG-04 / D-07)** — copy the existing `hasSelection` gate verbatim
(MessageActions.jsx:133): `{hasSelection && (<Tooltip ...><button onClick={handleReplace}.../></Tooltip>)}`.
So the focusable-button array is length 2 (Copy, Insert) or 3 (+ Replace).

**Green-check confirmation (D-06, 1.5 s)** — reuse `showConfirmation` + `CheckIcon`
(MessageActions.jsx:31-35, 50-54). 1500 ms timeout already present.

**Decision for the plan:** either (a) render `<MessageActions content={raw} hasSelection=…>` directly
inside the card (simplest, zero new handler code), or (b) export `handleCopy/handleInsert/handleReplace`
from `MessageActions.jsx` and call them from `FragmentCard` so the card owns its button refs for
keyboard nav. The research recommends (b) only if the kbd controller needs per-button refs — see the
keyboard section. Path (a) still works if refs are queried from the DOM (`getFocusables`-style).

### `Scribe/assistantSegments.js` (NEW — utility, pure transform)

**Analog:** `composeAssistantDisplay` (ChatMessageList.jsx:32-73) — lift the *exact same* REF-safe
placement, orphans-at-end, out-of-range-strip logic, but emit an ordered array of segment objects
instead of one composed string (a `<Card>` is a React node, not text).

**Placement source to copy** (ChatMessageList.jsx:41-72) — note: it uses `extractChannelMarkers`,
never a hand-rolled regex (CONTRACT-04 hazard):
```js
const hits = extractChannelMarkers(disc, 'fragment')   // ordered, 0-indexed, REF-safe
const referenced = new Set()
// ... substitution loop skips index < 0 || index >= frags.length, tracks referenced ...
composed = composed.replace(/\{\{fragment:(\d+)\}\}/g, (m, d) => {   // strip dangling out-of-range
  const idx = parseInt(d, 10)
  return idx < 0 || idx >= frags.length ? '' : m
})
const orphans = frags.filter((_, i) => !referenced.has(i))   // D-04: orphans at end
```

**Target shape** (segments, not string):
```js
// returns ordered [{type:'prose', md}, {type:'card', index, raw}, ...] ; orphan cards appended
export function buildAssistantSegments(discussion, fragments) {
  const disc = typeof discussion === 'string' ? discussion : ''
  const frags = Array.isArray(fragments) ? fragments : []
  if (frags.length === 0) return [{ type: 'prose', md: disc }]   // D-05/CONTRACT-02: NO cards
  const hits = extractChannelMarkers(disc, 'fragment')
  const valid = hits.filter(h => h.index >= 0 && h.index < frags.length)
  const referenced = new Set(valid.map(h => h.index))
  const segments = []; let cursor = 0
  for (const h of valid) {
    const marker = `{{fragment:${h.index}}}`
    const prose = disc.slice(cursor, h.position)
    if (prose) segments.push({ type: 'prose', md: prose })
    segments.push({ type: 'card', index: h.index, raw: frags[h.index] })
    cursor = h.position + marker.length
  }
  const cleanTail = disc.slice(cursor).replace(/\{\{fragment:(\d+)\}\}/g, (m, d) => {
    const i = parseInt(d, 10); return i < 0 || i >= frags.length ? '' : m
  })
  if (cleanTail) segments.push({ type: 'prose', md: cleanTail })
  frags.forEach((raw, i) => { if (!referenced.has(i)) segments.push({ type: 'card', index: i, raw }) })
  return segments
}
```
**Import:** `extractChannelMarkers` from `scribeResponse.js` (signature verified at scribeResponse.js:67-79;
`FRAGMENT_MARKER_RE` grammar at line 53). Never hand-roll `\{\{.+?\}\}` (would eat `{{REF:...}}`).

**Test analog:** mirror the pure-test style of `scribeResponse.spec.js` (no render) →
`assistantSegments.spec.js` covering CONTRACT-02 (0 frags → 1 prose segment), FRAG-01 (card at marker
position), D-04 (orphans at end), REF-safety (no `{{REF:...}}` consumed), out-of-range skip.

`getFocusables`-style cycle to copy (ScribeResultPanel.jsx:611-631):
```jsx
const handleKeyDown = useCallback(e => {
  const isNav = e.key === 'Tab' || e.key === 'ArrowLeft' || e.key === 'ArrowRight'
  if (!isNav) return
  e.preventDefault()
  const items = getFocusables()
  if (items.length === 0) return
  const currentIndex = items.indexOf(document.activeElement)
  const backward = e.shiftKey || e.key === 'ArrowLeft'
  const next = backward ? (currentIndex - 1 + items.length) % items.length
                        : (currentIndex + 1) % items.length
  items[next].focus()
}, [getFocusables])
```
Adapt for v3.1-04 keyboard reqs (a 2-axis coordinate over the thread):
- **KBD-02 (←/→):** cycle Copy/Insert/Replace on the focused card (the modulo pattern above, but
  **no wrap is not required for ←/→** — reqs say "cycle"; modulo wrap is acceptable).
- **KBD-03 (↑/↓):** move `cardPos` across all cards in thread order (oldest→newest). ↓ past the most
  recent card → `ChatInput.focus()`. **No wrap** upward past the oldest (unlike the modulo above —
  clamp, don't wrap).
- **KBD-01 (↑ from empty input):** focus the **last** card's Insert button; D-10 — if the last
  message is pure discussion (0 cards), walk backwards to the last message that has cards. No wrap.
- **KBD-04:** Escape → `ChatInput.focus()`; Enter/Space activate the focused `<button>` natively
  (intercept only arrows with `preventDefault`, like ScribeResultPanel.jsx:618).

**`useImperativeHandle` precedent** (ScribeActionMenu.jsx:81-85) — expose card button refs up to the
controller (each `FragmentCard` via `forwardRef`/`useImperativeHandle`), OR query buttons from the
card DOM node (`getFocusables` style). Either matches repo precedent.

**Auto-scroll unaffected:** the `useEffect` keyed on `messages.length` (lines 207-214) does not
re-fire when cards are added inside an existing message — fine, no change needed.

### `Scribe/MarkdownPreview.jsx` (MODIFY — preprocessor, transform)

**Analog:** the existing image-marker preprocessing stage (lines 134-141) — extend the SAME string
stage. It already enables `rehypeRaw` + `remarkGfm` (line 156), so raw HTML (`<sup>`) and GFM pipe
tables render.

**Current stage to extend** (lines 134-141):
```jsx
const preprocessed =
  typeof children === 'string'
    ? children.replace(/\{\{IMG:(scribe-img-\d+)\}\}/g, '![IMG:$1](placeholder)')
    : children
```
Add the D-02 transforms (cosmetic ONLY — returns a NEW string; the caller's `raw` is never mutated):
```jsx
// order: tables first (multiline blocks), then inline markers
if (out.includes('[TABLE:') || out.includes('[CELL:')) {
  out = transformCellMarkersForPreview(out, '').displayMd   // {displayMd, warning}; enrichedMd '' → no count validation
}
out = out.replace(/\{\{REF:scribe-ref-\d+:([^}]*)\}\}/g, '$1')          // REF → visible text only
out = out.replace(/\{\{IMG:(scribe-img-\d+)\}\}/g, '![IMG:$1](placeholder)')  // existing image step
out = out.replace(/\[\^scribe-fn-(\d+)\]/g, '<sup style="opacity:0.5">$1</sup>')  // footnote → discreet (rehype-raw renders it)
```

**`transformCellMarkersForPreview` signature (verified, tableCellMarkers.js:168):**
`transformCellMarkersForPreview(md, enrichedMd) → { displayMd, warning }`. With `enrichedMd === ''`
it returns `{ valid:true }` internally (no count validation) — safe to pass `''` (verified lines
181-183, 209-211). Gate the call on `includes('[TABLE:'|'[CELL:')` so an already-converted pipe-table
(e.g. from the popover path `ScribePopover.jsx:180`) is left untouched → **idempotent** (Open Question
2). The existing `img` component override (lines 106-131) already renders the `![IMG:...]` placeholder
chip — keep it.

**Pitfall (GATE §8 flaky fragments):** unmatched/missing markers must pass through visibly, never
throw. A fragment with no markers renders as plain markdown.

**Security note (V5):** keep preprocessor output to fixed HTML around a captured `\d+` only — never
interpolate model-controlled substrings into an HTML tag/attribute. `rehypeRaw` XSS exposure is
**pre-existing** (popover already uses it); do NOT regress escaping; any `rehype-sanitize` hardening
is a human decision flagged for v3.1-05.

### Scribe-purple accent (D-01)

### Theme-aware bubble styling
**Source:** `AssistantBubble` (ChatMessageList.jsx:140-159) + `MarkdownPreview` (`useTheme`,
`isDark = (theme.palette.type || theme.palette.mode) === 'dark'`).
**Apply to:** `FragmentCard` surface, any new container — derive light/dark from `useTheme()`.

### panelActions bridge (insert/replace) — pass RAW only
**Source:** `View.jsx:214-246` wires `setPanelActions({ insert: handleInsert, replace: handleReplace })`.
`handleInsert(text)` does `markdownToHtml(text)` + `castPanelAction(...)` → OO plugin parses markers.
**Apply to:** every card action — `panelActions.insert(raw)` / `panelActions.replace(raw)`. The
markdown→HTML and rich reinjection are downstream and **unchanged** (FRAG-03).

### Manual-index keyboard navigation (no library)
**Source:** `ScribeResultPanel.jsx:587-631` (flat focusables + modulo cycle) and
`ScribeActionMenu.jsx:43-44, 81-85, 96-103, 136-163` (focusIndex state, `useImperativeHandle`,
`updateFocus`, `onArrow` plumbing).
**Apply to:** the `ChatMessageList` thread-level controller and `ChatInput` ↑ hook. Do NOT add a
roving-tabindex dependency (zero such deps in repo).

### i18n labels (already present)
**Source:** `MessageActions.jsx:113, 123, 134` — `t('Scribe.button.copy'|'insert'|'replace')`.
**Apply to:** card buttons reuse the same keys. Full 5-locale fallback-message i18n is v3.1-05.

### Clipboard dual-format + 1.5 s green check
**Source:** `MessageActions.jsx:50-72` (ClipboardItem text/html + text/plain, `showConfirmation` 1500 ms,
`stripParagraphWrapper` 37-41, `markdownToHtml` import line 8).
