# Phase v3.1-04 Plan 04: Rendu chat — segment-based assistant render Summary

Rewrote the `ChatMessageList` assistant branch to render one bubble that interleaves discussion
prose (`MarkdownPreview`) with fragment cards (`FragmentCard`) at their `{{fragment:N}}` positions,
deleting the throwaway `composeAssistantDisplay` and removing the message-level `MessageActions`
(D-08) so actions live only on cards.

## What was built

- **Task 1 (RED):** `ChatMessageList.render.spec.jsx` — RTL coverage for FRAG-01 marker placement,
  D-04 orphans, CONTRACT-02/D-05 (0-fragment → discussion only, zero action UI), no-empty-bubble,
  D-08 (actions only on cards; Replace gated on selection), backward-compat legacy prose bubble,
  user/error branches, and D-03 raw-fragment passthrough to `panelActions.insert`. 13 tests, RED
  before implementation.
- **Task 2 (GREEN):** `ChatMessageList.jsx` rewritten:
  - Deleted `composeAssistantDisplay`; dropped `extractChannelMarkers`, `MessageActions`, and bare
    `react-markdown` imports.
  - Added `buildAssistantSegments`, `FragmentCard`, `MarkdownPreview` imports.
  - `AssistantBubble` now renders prose via `MarkdownPreview` (cosmetic marker cleanup + theme).
  - Contract assistant messages map `buildAssistantSegments(msg.discussion ?? msg.content, msg.fragments)`:
    prose → `MarkdownPreview`, card → `FragmentCard raw=seg.raw hasSelection={!!currentSelection}`.
  - No-empty-bubble guard returns `null` for a single empty prose segment.
  - Legacy messages render a single prose `AssistantBubble`, no actions (D-08).
  - Message-level `MessageActions` removed entirely from the assistant branch (D-08).
