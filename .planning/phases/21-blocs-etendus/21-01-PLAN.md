**1. flattenTokens — add two new block type handlers before the fallback else-if chain (around line 161, after the "space" handler):**

A) Handle `code` tokens (fenced code blocks from marked.lexer):
```javascript
} else if (block.type === "code") {
  // Fenced code block: split by newlines, each line = one code_block block
  var codeLines = (block.text || "").split("\n");
  for (var cl = 0; cl < codeLines.length; cl++) {
    blocks.push({
      type: "code_block",
      runs: [{ text: codeLines[cl] || " ", bold: false, italic: false, strikethrough: false, code: true, link: null }]
    });
  }
```
Note: empty lines get a space character " " so OO creates a visible paragraph (empty paragraphs with shading look wrong).

B) Handle `blockquote` tokens (blockquotes contain nested block tokens):
```javascript
} else if (block.type === "blockquote") {
  // Blockquote: recursively flatten inner tokens, tag each as blockquote
  var innerBlocks = flattenTokens(block.tokens || []);
  for (var bq = 0; bq < innerBlocks.length; bq++) {
    innerBlocks[bq].blockquote = true;
    blocks.push(innerBlocks[bq]);
  }
```
This recursion reuses the existing flattenTokens logic for the blockquote's inner content (paragraphs, headings, lists, even nested blockquotes). The `blockquote: true` flag tells the builder to apply indentation.

    r.SetFontFamily("Courier New");
    if (srcFontSize) r.SetFontSize(srcFontSize);
