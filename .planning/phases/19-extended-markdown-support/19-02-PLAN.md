<task type="auto">
  <name>Task 2: Extend callCommand run loops for strikethrough, code, and hyperlinks</name>
  <files>plugins/onlyoffice-scribe/scripts/code.js</files>
  <action>
Modify the run loops inside the callCommand interpreter (inside buildAndInject's callCommand callback) to handle strikethrough, code, and link properties on runs. All code must be ES5.

The callCommand interpreter has run loops in three block type handlers (heading, list_item, paragraph). Each currently creates ApiRun with SetBold/SetItalic. Replace ALL THREE run loops with a shared pattern that handles the new run properties.

**For each run loop** (in heading, list_item, and paragraph blocks), replace the inner loop body with:

```javascript
var runs = block.runs || [];
for (var j = 0; j < runs.length; j++) {
  var run = runs[j];
  if (run.link) {
    var link = Api.CreateHyperlink(run.link, run.text, "");
    p.AddElement(link);
  } else {
    var r = Api.CreateRun();
    r.AddText(run.text);
    if (run.bold) r.SetBold(true);
    if (run.italic) r.SetItalic(true);
    if (run.strikethrough) r.SetStrikeout(true);
    if (run.code) {
      r.SetFontFamily("Courier New");
      // Only apply srcFontSize for code spans, NOT srcFontFamily
      if (srcFontSize) r.SetFontSize(srcFontSize);
    } else if (block.type !== "heading") {
      // Apply source font to non-heading, non-code runs
      if (srcFontFamily) r.SetFontFamily(srcFontFamily);
      if (srcFontSize) r.SetFontSize(srcFontSize);
    }
    // Heading runs: no srcFont applied (heading style defines sizing)
    p.AddElement(r);
  }
}
```

**Key points from research (DO NOT deviate):**
- Hyperlinks use `Api.CreateHyperlink(url, text, tooltip)` + `p.AddElement(link)` -- NOT a regular run with blue color (Pitfall 5)
- Code spans use `r.SetFontFamily("Courier New")` -- do NOT apply srcFontFamily (it would override monospace)
- Code spans DO apply srcFontSize to keep size consistent with surrounding text
- Heading runs get NO srcFontFamily and NO srcFontSize (Pitfall 3 -- heading style defines sizing)
- Strikethrough uses `r.SetStrikeout(true)` -- the method name is SetStrikeout, NOT SetStrikethrough
- All code inside callCommand must be ES5 (var only, no arrow functions, no template literals)
- The `block.type !== "heading"` check controls font application: headings get no font override, others do
