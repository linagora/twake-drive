# Phase v3.1 Plan 06-02: Regression corpus + response_format decision Summary

A static, network-free malformed-response regression corpus drives deterministic Jest assertions over `parseScribeResponse` for all six D-07 categories; the ~2% code-fence `fellBack` reliquat is re-measured at 0% (confirmed closed by the existing `stripFence`, no change), and the `response_format` json_object default is documented with the now-confirmed cozy-stack proxy-forwarding fact.

## What Was Built

### Task 1 — Static fixtures + deterministic corpus spec (commit b9333ae3b)
- `scribeResponse.fixtures.js` exports `SCRIBE_RESPONSE_FIXTURES`, a plain object keyed by exactly the six D-07 categories — `fence`, `preamble`, `trailingComma`, `splitTable`, `brokenRef`, `nonJsonProse` — each a non-empty array of `{ name, raw, surface, expect }` static literals (English + French). No network, no model, no GATE-harness import.
- `scribeResponse.corpus.spec.js` imports the fixtures + `parseScribeResponse` and, via `describe.each` / `it.each`, asserts each fixture's documented `fellBack`/`valid` outcome plus optional `discussion`/`fragments`/`warningsInclude`/`fragmentsContain`. Dedicated blocks assert `splitTable` → `valid:false` + `warnings` contains `'split-table'`, and `nonJsonProse` → surface-specific fallback (popover `fragments=[raw]`, chat `discussion=raw`). Every category array asserted non-empty.

### Task 2 — Fence fellBack re-measure + response_format decision (commit 9f1c29151)
- Added a `fellBack rate` describe block to the corpus spec that asserts every `fence`-category fixture yields `fellBack===false` (0% fallback), including the three required shapes (language-tag, trailing-whitespace, no-trailing-newline) and a **verbatim real-reliquat sample** matching STATE.md's observed `\`\`\`json`-wrapped output.
- Re-measure outcome: **0% fallback — the reliquat is already closed by the existing `stripFence`**. No extension was required. A confirming comment (no behavior change) was added at the `stripFence` definition in `scribeResponse.js`; the ReDoS contract is preserved (anchored/linear regexes, grep gate clean).
- `RESPONSE_FORMAT_DECISION.md` documents: the `json_object` default (scribeAI.js l.279); the **confirmed** proxy fact — cozy-stack `CallRAGQuery` (`chat.go` ~l.446-464) forwards the POST body **unchanged** via `bytes.NewReader(payload)`, so `response_format` (and a future `json_schema`) reach the upstream model; the `json_schema` deferral is therefore a model-capability question (does Mistral-Small-3.2-24B honor it?), not a proxy limitation; `SCRIBE_OUTPUT_SCHEMA` is the ready payload; and the auditable table of re-measured fence shapes.
