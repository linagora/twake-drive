# Phase v3.1-06 Plan 01: Re-ask Shared Helper Summary

A single shared `callScribeAIWithReask` helper that, on a parse-invalid first LLM response (`fellBack || !valid`), re-solicits the model exactly once with a corrective nudge before the existing contextual fallback applies (HARDEN-01) — wired through BOTH the chat (`ScribeContext.jsx`) and popover/inline (`ScribePopover.jsx`) call sites so the two surfaces cannot diverge, with the caller's abort signal forwarded to both attempts.

## What Was Built

- **`scribeAI.js::callScribeAIWithReask(client, messages, { signal, surface })`** — awaits `callScribeAI` once and parses via `parseScribeResponse(text, { surface })`; if the first parse is clean (`valid && !fellBack`) it returns it with NO second call; otherwise it builds `messages.concat([{ role: 'user', content: REASK_CORRECTION_NUDGE }])` (non-mutating), awaits a single re-ask, and returns that parse unconditionally (hard cap = 1 retry). The same `signal` is forwarded to both `callScribeAI` calls; `AbortError`/`FetchError` propagate unchanged. No loading/telemetry side effects live inside the helper. Imports `parseScribeResponse` from `scribeResponse.js`.
- **`scribeAI.js::REASK_CORRECTION_NUDGE`** — a terse, English instruction-channel string telling the model its previous reply was not valid contract output and to reply ONLY with a `discussion`/`fragments` JSON object (no prose, no fences, insertable text only in `fragments`, balanced `[TABLE:N]`/`[CELL:r,c]` markers). Consistent with `RESPONSE_CONTRACT_CORE`'s separation imperative; does not restate the whole contract.
- **`scribeAI.spec.js`** — 8 new tests under `describe('callScribeAIWithReask …')` using a `fetchJSON`-mocking fake client: clean first response → 1 call; `fellBack` first → 2 calls; `valid===false` (split table) first → 2 calls; second invalid → still 2 (never 3); retry appends exactly one `REASK_CORRECTION_NUDGE` message (`secondMsgs.length === firstMsgs.length + 1`) without mutating the original; same `signal` forwarded to both calls and `AbortError` propagates; surface honored on the final fallback parse; nudge is a JSON-mentioning string.
- **`ScribeContext.jsx` (chat)** — collapsed `callScribeAI(client, aiMessages)` + `parseScribeResponse(responseText, { surface: 'chat' })` into one `const parsed = await callScribeAIWithReask(client, aiMessages, { surface: 'chat' })`. Migrated EVERY `responseText` consumer to `parsed.raw` (the unconditional `logScribeExchange({ rawResponse })` and the dev-guarded `devExchange.rawResponse`). Dropped the now-unused `callScribeAI` and `parseScribeResponse` imports (kept `serializeAssistantTurnForHistory`). Chat holds no AbortController today, so no signal is passed — behavior preserved.
- **`ScribePopover.jsx` (popover/inline)** — collapsed `callScribeAI(client, messages, { signal: controller.signal })` + `parseScribeResponse(text, { surface: 'popover' })` into one `const parsed = await callScribeAIWithReask(client, messages, { signal: controller.signal, surface: 'popover' })`; `setRawResponse(text)` → `setRawResponse(parsed.raw)`. Updated the stale "Re-ask on invalid parse is deferred to v3.1-05." comment to state the re-ask now happens upstream in the helper. Dropped the now-unused `callScribeAI` and `parseScribeResponse` imports. `recordProbeSample` / `logScribeExchange` / `setParsedResponse` / single-fragment normalization unchanged, operating on `parsed`.

## Threat Surface

All three mitigatable threats from the plan's threat register are satisfied by construction:
- **T-v3.1-06-01 (DoS, retry loop)** — hard cap of exactly one re-ask; the second parse is returned unconditionally with no recursion / no `while`. Asserted by the call-count spec ("never 3 calls").
- **T-v3.1-06-02 (Tampering, re-ask text)** — the second response flows through the SAME `parseScribeResponse` (prototype-pollution-safe staging, ReDoS-safe regexes, MAX_RAW_LENGTH guard); the helper introduces no new parse path.
- **T-v3.1-06-03 (DoS, abort on 2nd call)** — the caller's `signal` is forwarded to both calls and `AbortError` propagates (not swallowed). Asserted by the abort spec.

No new security-relevant surface introduced (no new endpoints, no package installs).
