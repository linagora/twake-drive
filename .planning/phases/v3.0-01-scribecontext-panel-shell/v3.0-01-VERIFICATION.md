### Human Verification Required

#### 1. Panel open/close with editor resize

**Test:** Open an OO document with `drive.scribe.enabled` flag enabled. Click the panel toggle button in the floating zone (bottom-right, panel icon).
**Expected:** A 400px panel appears on the right side. The OO editor visibly shrinks horizontally. The floating buttons disappear while the panel is open.
**Why human:** Flex layout rendering and actual iframe pixel-level resize cannot be verified by static code inspection.

#### 2. Panel close button restores editor width

**Test:** With the panel open, click the X button in the panel header.
**Expected:** Panel closes. OO editor expands back to full width. Floating zone reappears.
**Why human:** Layout reflow and focus restoration (setTimeout 100ms) require live browser rendering.

#### 3. localStorage persistence across reloads

**Test:** Open panel, reload the page.
**Expected:** Panel reopens because `localStorage.getItem('scribe-panel-open') === 'true'`.
**Why human:** Requires browser reload to confirm state restoration.

#### 4. Ctrl+Shift+I from open popover opens panel

**Test:** Select text, trigger the inline Scribe popover (Ctrl+Shift+I). With popover open, press Ctrl+Shift+I once more.
**Expected:** Popover closes, panel opens.
**Why human:** Depends on cross-frame keyboard event propagation and timing of the 200ms popover-ready delay in View.jsx.

#### 5. Double Ctrl+Shift+I from editor toggles panel

**Test:** With no text selected, press Ctrl+Shift+I twice within 350ms.
**Expected:** Panel toggles open on the second press.
**Why human:** Timing-sensitive double-tap detection in code.js (editDelayTimer logic) requires live interaction to validate.

#### 6. Inline Scribe popover regression check

**Test:** With panel closed and text selected, trigger Ctrl+Shift+I. Use the popover normally to get an AI response, then use Replace or Insert.
**Expected:** Full popover flow works exactly as before this phase — no regressions.
**Why human:** End-to-end AI call and UI interaction requires live environment.
