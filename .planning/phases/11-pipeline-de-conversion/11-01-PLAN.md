**GREEN phase:** Create `scribeConversion.js` following the research pattern exactly:

1. `normalizeOoHtml(html)` internal function:
   - Uses `DOMParser` to parse HTML
   - Converts `<span style="font-weight:bold">` to `<strong>` (check both "bold" string and numeric >= 700)
   - Converts `<span style="font-style:italic">` to `<em>`
   - Handles combined bold+italic in single span: wrap as `<strong><em>content</em></strong>`
   - Process bold+italic together in one pass per span (not two separate passes) to avoid double-processing
   - Returns `doc.body.innerHTML`

2. `createTurndownService()` internal function:
   - Configure: `headingStyle: 'atx'`, `bulletListMarker: '-'`, `codeBlockStyle: 'fenced'`, `emDelimiter: '*'`
   - Apply GFM plugin: `td.use(gfm)`
   - Remove unsupported: `td.remove(['img', 'svg', 'math', 'object', 'embed', 'iframe', 'script', 'style'])`
   - Add `blankParagraph` rule: detect `<p>` with empty/whitespace/nbsp content, return `'\n\n'`

3. `htmlToMarkdown(html)` exported:
   - Guard: if `!html || !html.trim()` return `''`
   - Run `normalizeOoHtml(html)` then `createTurndownService().turndown(normalized)`

4. `markdownToHtml(markdown)` exported:
   - Guard: if `!markdown || !markdown.trim()` return `''`
   - Return `marked.parse(markdown)` (GFM enabled by default)

**REFACTOR phase:** Clean up if needed. Ensure all tests pass.

Note on test environment: Tests run in jsdom (jest default) which provides DOMParser. If DOMParser is unavailable in the test env, mock it or configure jest with jsdom environment explicitly.
