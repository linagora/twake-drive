# Phase v3.1-05 Plan 01: Popover result rendered as a Scribe-violet card — Summary

Popover `result` step now renders the single normalized fragment inside a Scribe-violet (#7C3AED) card matching the chat FragmentCard, fed the RAW fragment through MarkdownPreview so markers (REF/TABLE/footnote) are cleaned for display only while Insert/Replace/Retry keep operating on the RAW string.

## What was built

- **`ScribeResultPanel.jsx`**
  - Added module constants `SCRIBE_PURPLE = '#7C3AED'` and `SCRIBE_PURPLE_08`, aligned with `FragmentCard` / `ChatMessageList`.
  - Added a `rawFragment` prop (string, defaults to `''`); documented as the RAW normalized fragment (markers intact).
  - Wrapped the non-dev `resultContent` body in a violet card frame (`border: 1px solid SCRIBE_PURPLE`, dark-aware bg `rgba(124,58,237,0.12)` / `SCRIBE_PURPLE_08`, `borderRadius: 8`, `padding: '8px 10px'`) tagged `data-scribe-result-card`.
  - Card body now renders `<MarkdownPreview>{rawFragment}</MarkdownPreview>` instead of `{resultText}`, so MarkdownPreview performs the single cosmetic marker cleanup (matching the chat card).
  - `cellWarning` banner kept above the card; `error` path, the dev-mode `ScribeDevPanels` branch, and the Insert/Replace/Retry footer all preserved unchanged.
  - Updated propTypes/defaultProps for `rawFragment`.
- **`ScribePopover.jsx`**
  - Passes `rawFragment={rawResult}` (RAW normalized fragment from the l.240-261 normalization) to `<ScribeResultPanel>` for the cosmetic card render.
  - `handleInsert` / `handleReplace` unchanged — still route on `rawResult || result.text` (RAW-correct, D-02).
  - Normalization and the `addMessage` chat mirror (INLINE-02 wiring) untouched; `parsed.discussion` is never rendered in the popover (only used inside the chat mirror), satisfying D-03 / INLINE-01.
- **`ScribeResultPanel.spec.jsx`** (new, TDD): asserts the violet card frame, RAW→cosmetic display (REF marker hidden, "Fig 1" visible, cosmetic `resultText` NOT shown), empty/whitespace fragment safety, and preserved footer / cellWarning / error surfaces.
