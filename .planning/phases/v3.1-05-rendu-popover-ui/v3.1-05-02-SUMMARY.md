# Phase v3.1-05 Plan 02: Action-menu rework Summary

**Free-prompt folded into the popover action list as an integrated entry, with a bottom "Ouvrir le panneau latéral" ListItem replacing the removed in-popover icon button, and keyboard roving extended to actions -> prompt -> open-panel.**

## Performance

- **Duration:** ~18 min
- **Started:** 2026-06-22 (worktree execution)
- **Completed:** 2026-06-22
- **Tasks:** 1 of 1 implementation task (Task 2 is a deferred human-verify checkpoint — see below)
- **Files modified:** 2 modified, 1 created

## Accomplishments
- Relocated the `ScribePromptInput` from a detached `<Paper width:500>` card into the menu `<Paper>` as an integrated row at `PROMPT_INDEX`, preserving its focus highlight and `updateFocus(PROMPT_INDEX)` hover wiring.
- Removed the standalone in-popover open-panel `<button>` + tooltip (former l.433-480) and replaced it with a bottom `<ListItem>` calling the same `onOpenPanel`, reusing the existing `Scribe.button.open_panel` i18n key (D-06).
- Added `OPEN_PANEL_ENTRY` to `scribeActions.js` and injected it at render time (DEV_MD_ACTION precedent) so it is never a `SCRIBE_ACTIONS` member — no LLM-pipeline leak (T-v3.1-05-04).
- Extended keyboard roving to a single `rovingCount` modulus (actions -> prompt -> open-panel -> wrap); `handlePromptArrow('down')` now targets the open-panel entry, and Enter/Space on the open-panel index calls `onOpenPanel`.
- Left `ScribeFloatingButton.jsx` (floating document PanelIcon) and `ScribePopover.jsx` untouched (D-07; `onOpenPanel` threading already existed).
- Added `ScribeActionMenu.spec.jsx` (9 tests) covering render order, click -> `onOpenPanel`, the no-`onSelect`/pipeline-leak guard, free-prompt id preservation, and roving (down-cycle, Enter-on-open-panel, up-wrap).

## Decisions Made
- **Prompt stays a dedicated `ScribePromptInput`, relocated into the list** (Claude's Discretion per D-05): keeps the `onSelect('free-prompt', prompt, prompt)` call and the input's own arrow contract verbatim, avoiding a `type:'input'` rewrite.
- **`OPEN_PANEL_ENTRY` lives in `scribeActions.js` but is injected in `ScribeActionMenu`** (not in `SCRIBE_ACTIONS`): satisfies the T-v3.1-05-04 boundary while keeping the entry's id/labelKey declarative and reusable.
- **Open-panel entry guarded by `onOpenPanel` presence**: when absent, the prompt entry becomes last and takes the `'0 0 8px 8px'` corners; when present, the open-panel entry takes them.
