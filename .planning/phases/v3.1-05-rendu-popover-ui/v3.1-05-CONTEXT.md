<decisions>
## Implementation Decisions

### Rendu du résultat popover en carte
- **D-01:** Le panneau résultat du popover (étape `result` de `ScribePopover`) rend le fragment unique **en carte**, aligné sur la `FragmentCard` du chat (v3.1-04) : encadrement accent violet Scribe, markdown riche via `MarkdownPreview`, marqueurs `{{REF:…}}`/`[TABLE:N]`/`[^scribe-fn-N]` rendus **proprement** (D-02 de v3.1-04). Réutiliser `FragmentCard`/`MarkdownPreview` plutôt que dupliquer.
- **D-02:** INVARIANT D-03 de v3.1-04 conservé : l'affichage propre est **cosmétique** ; Insérer/Remplacer/Copier opèrent sur le **fragment BRUT** (marqueurs intacts) — le pipeline de réinjection riche l'exige. La normalisation au fragment unique (`ScribePopover` l.240-250) reste la source du `rawFragment`.
- **D-03:** Le popover **n'affiche jamais la `discussion`** (INLINE-01) — seul le fragment normalisé est rendu. La discussion est déjà miroitée dans l'historique chat (INLINE-02, câblage v3.1-02) ; cette phase **valide** ce miroir de bout en bout sans le ré-implémenter.

### Refonte du menu d'actions (popover)
- **D-04:** Ordre de la liste du menu (validé) : `Corriger la grammaire`, `Traduire ▸`, `Changer le ton ▸`, `Améliorer ▸`, puis **l'input prompt libre comme entrée intégrée**, puis l'entrée **« Ouvrir le panneau latéral »** tout en bas. (Schéma validé en discussion.)
- **D-05:** Le **prompt libre** n'est plus un champ séparé sous le menu (carte distincte actuelle) → il devient une **entrée/input dans la liste** du menu (`SCRIBE_ACTIONS` / `ScribeActionMenu`), navigable au clavier comme les autres entrées.
- **D-06:** L'**icône open-panel DANS le popover** (prop `onOpenPanel` passée au menu, `ScribePopover.jsx` l.423) est **retirée** et remplacée par l'entrée de menu « Ouvrir le panneau latéral », qui appelle le même `onOpenPanel`. Réutiliser la clé i18n existante `Scribe.button.open_panel`.
- **D-07:** Le **bouton flottant** séparé `PanelIcon` (`ScribeFloatingButton`, `onTogglePanel`, posé sur le document, hors popover) est **conservé** — accès direct au panneau maintenu, pas de redondance jugée gênante.

### Claude's Discretion
- Mécanique clavier exacte pour la nouvelle entrée prompt + l'entrée « ouvrir panneau » dans la liste (l'input prompt est déjà géré à l'index `n` ; l'entrée panneau s'ajoute après — laisser le planner confirmer l'indexation/roving).
- Réutilisation directe de `FragmentCard` vs petit wrapper popover-spécifique (le popover n'a ni positionnement `{{fragment:N}}` ni multi-cartes — un seul fragment), tant que D-01/D-02 tiennent.
- Sémantique de l'entrée « Ouvrir le panneau latéral » (ouvrir simplement via `onOpenPanel` existant ; report éventuel de la sélection/prompt courant = comportement existant, non élargi ici).
- Détails de style (réutiliser `scribe.styl` / thème clair-sombre) tant que la cohérence visuelle avec les cartes du chat est respectée.
</decisions>

## Existing Code Insights

### Reusable Assets
- `FragmentCard` + `MarkdownPreview` (v3.1-04) : rendu carte markdown riche + marqueurs propres + actions sur fragment brut → réutilisables tels quels pour le fragment unique du popover.
- `ScribePopover` normalise déjà au **fragment unique** (l.240-250 : join si 2+, promotion discussion si 0, 1:1 sinon) → fournit le `rawFragment`.

- Clé i18n `Scribe.button.open_panel` déjà définie dans les 14 locales.
- `ScribeActionMenu` gère déjà un input (prompt libre) à l'index `n` au clavier → l'intégration comme entrée de liste s'appuie sur cette mécanique.

- Popover = machine à états `menu` → `loading` → `result` (`ScribePopover.jsx`).
- Actions branchées sur `panelActions`/bridge OO → pipeline de réinjection riche (inchangé ; le fragment passé doit rester **brut**).
- Accent violet Scribe `#7C3AED` + thème clair/sombre via `useTheme()` / `scribe.styl`.

### Integration Points
- `ScribePopover` (result) ↔ `FragmentCard`/`MarkdownPreview` (rendu carte du fragment unique).
- `ScribeActionMenu`/`scribeActions.js` ↔ `onOpenPanel` (nouvelle entrée) ; input prompt intégré à la liste.
- Insertion/Remplacement → callbacks existants sur le fragment **brut** normalisé.
