## Tests

### 1. Send a message and see it in chat
expected: Open the Scribe panel. Type a message in the textarea at the bottom, press Enter (or click the purple send button). Your message appears as a user bubble in the conversation above.
result: pass

### 2. AI responds with Markdown rendering
expected: After sending a message, an AI response appears below your message. If the response contains formatting (bold, lists, code blocks, tables), it is rendered as rich Markdown — not raw text.
result: pass

### 3. Conversation scrolls correctly
expected: Send several messages to build a long conversation. The chat auto-scrolls to show the latest message. You can scroll up to see earlier messages. No overflow issues — the chat scrolls within the panel, not the whole page.
result: pass

### 4. Multi-turn context maintained
expected: Send a first message (e.g., "My name is Ben"). Then send a follow-up (e.g., "What's my name?"). The AI should remember context from previous messages and respond correctly.
result: pass

### 5. Loading indicator while AI responds
expected: After sending a message, while waiting for the AI response, you see a spinner with "Scribe is thinking..." (or the French equivalent) below your message. It disappears when the response arrives.
result: pass

### 6. Error appears as chat message
expected: Trigger an error (e.g., disconnect network briefly, or send while API is unreachable). The error appears as a message in the chat — not a popup or crash. The chat remains usable.
result: pass

### 7. Popover actions appear in panel history
expected: With the panel open, close it. Select text in OO, use the inline Scribe popover to run an action (e.g., Reformulate). Open the panel again — the popover action (breadcrumb + AI result) appears in the conversation history.
result: pass

### 8. Shift+Enter inserts newline
expected: In the chat textarea, press Shift+Enter. A newline is inserted without sending the message. Press Enter alone to send.
result: pass
