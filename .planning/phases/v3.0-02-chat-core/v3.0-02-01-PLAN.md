**ScribeContext.jsx** -- extend the existing provider with conversation state:

1. Add state: `messages` (array), `isLoading` (boolean). Message shape: `{ id, role, content, timestamp }` where role is 'user' | 'assistant' | 'error'.

2. Add `sendMessage(text)` callback that:
   - Creates a user message `{ id: Date.now(), role: 'user', content: text, timestamp: new Date() }` and appends to messages
   - Sets isLoading = true
   - Builds the AI messages array: a system message with a CHAT system prompt (different from the inline SYSTEM_PROMPT -- this one should say "You are a helpful writing assistant. Help the user with their writing tasks. Respond in the same language as the user's message. Use Markdown formatting when appropriate."), followed by all conversation messages mapped to `{ role, content }` (skip 'error' messages)
   - Calls `callScribeAI(client, aiMessages)` (import callScribeAI and classifyScribeError from scribeAI.js)
   - On success: appends assistant message `{ id: Date.now()+1, role: 'assistant', content: responseText, timestamp: new Date() }`
   - On error: appends error message `{ id: Date.now()+1, role: 'error', content: t(classified.messageKey), timestamp: new Date() }`
   - Sets isLoading = false in finally block
   - NOTE: `client` comes from `useClient()` imported from cozy-client, called inside ScribeProvider

3. Add `addMessage(msg)` callback for the popover refactoring (plan 02) -- simply appends a message object to the messages array.

4. Export the extended value: `{ isPanelOpen, togglePanel, openPanel, closePanel, messages, isLoading, sendMessage, addMessage }`

**ChatMessageList.jsx** -- new component:

1. Import ReactMarkdown from 'react-markdown' (ESM default import)
2. Accepts no props -- reads `messages` and `isLoading` from `useScribe()`
3. Renders a scrollable container (flex: 1, overflowY: 'auto', padding: 16px)
4. Maps over messages to render bubbles:
   - User messages: right-aligned, background `theme.palette.action.selected`, plain text, border-radius 12px (top-right: 4px)
   - Assistant messages: left-aligned, no background (or very subtle), rendered via `<ReactMarkdown>{msg.content}</ReactMarkdown>`, border-radius 12px (top-left: 4px)
   - Error messages: left-aligned, red-tinted background (`theme.palette.error.main` at 10% opacity), text in error color, small warning icon or prefix
5. Auto-scroll: use a `useEffect` with a ref on the container -- when `messages.length` changes or `isLoading` changes, scroll to bottom via `containerRef.current.scrollTo({ top: containerRef.current.scrollHeight, behavior: 'smooth' })`
6. When `isLoading` is true, show a typing indicator at the bottom (three animated dots or a small Spinner from cozy-ui with "..." text)
7. Style ReactMarkdown output: ensure code blocks have background/padding, tables have borders, lists have proper indentation. Use inline styles on a wrapper div with className approach or style the markdown container. Keep it simple -- a wrapper div with `style={{ fontSize: 14, lineHeight: 1.5, wordBreak: 'break-word' }}` and let browser defaults handle most markdown elements.
8. When messages is empty, show a centered welcome message: the existing sparkle SVG + tagline (move from ScribePanel or duplicate)

Add i18n keys to en.json and fr.json under "Scribe.chat":
