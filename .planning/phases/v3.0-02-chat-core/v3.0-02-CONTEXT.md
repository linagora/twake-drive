# Phase v3.0-02: Chat Core - Context

**Gathered:** 2026-03-15
**Status:** Ready for planning
**Source:** User discussion

<domain>
## Phase Boundary

Build a working multi-turn chat in the ScribePanel. The user types messages, the AI responds, and the full conversation is visible in the panel. The popover's Scribe actions (rephrase, summarize, etc.) also feed into the same conversation history, so the panel shows everything.

</domain>

<decisions>
## Implementation Decisions

### State Management
- Conversation state (messages[], sendMessage, isLoading) lives in ScribeContext (option A)
- Conversation persists across panel open/close within the session
- Popover actions write to the same message history as the panel chat
- When user opens the panel, they see all previous interactions (including those triggered from the popover)

### API Calls
- Simple (no streaming): wait for full response then display
- Reuse existing callScribeAI from scribeAI.js, adapted for multi-turn (full message history sent each time)
- No retry button for now (deferred)

### Markdown Rendering
- Use react-markdown for rendering AI responses (not marked.parse)
- Read-only rendering for now (no contentEditable — deferred from this milestone entirely)

### Chat Input
- Enter = send message, Shift+Enter = new line
- Send button visible
- Input disabled while AI is loading

### Chat UX
- Auto-scroll to latest message
- User can scroll up to see earlier messages
- Loading indicator while AI responds
- Errors appear as chat messages (no retry button)

### Panel Layout
- Keep current panel width (400px) — resize is phase v3.0-04
- Keep existing header (Scribe + close button)
- No "new chat" / "clear conversation" button for now

### Popover Refactoring
- ScribePopover currently calls callScribeAI internally
- Must be refactored so actions go through ScribeContext to write messages to shared history
- Popover keeps its own UI flow (menu → loading → result) but the underlying data flows through the shared conversation

### Claude's Discretion
- Message bubble styling and layout
- How popover actions are represented in the chat history (e.g., system message format)
- Scroll behavior implementation details
- How conversation is cleared (currently: only on page reload)

</decisions>

<specifics>
## Specific Ideas

- AI responses should render markdown with bold, lists, code blocks, tables via react-markdown
- User messages displayed as-is (plain text)
- Each message object should have: role (user/assistant/system), content, timestamp
- The popover "breadcrumb" (e.g., "Reformuler > Anglais") could serve as the user message when an action comes from the popover

</specifics>

<deferred>
## Deferred Ideas

- ContentEditable editing of AI responses — removed from this milestone entirely
- Streaming responses — not for this phase
- Retry button on errors — later
- Multiple conversations / "new chat" — later
- Panel resize — phase v3.0-04

</deferred>
