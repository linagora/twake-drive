# Phase 24.1: Table round-trip rearchitecture — clone + InsertContent

**Gathered:** 2026-03-23
**Status:** Ready for planning
**Source:** In-session design discussion after Phase 24 implementation review

<domain>
## Phase Boundary

Rearchitect the table round-trip to use clone + InsertContent instead of in-place modification. This fixes Insert mode (which was broken) and enables mixed text+table content handling.

**Why this exists:** Phase 24 implemented table round-trip via in-place cell modification (`cell.Clear()` + rebuild). This approach:
- Breaks Insert mode entirely (no table is created for the inserted copy)
- Requires a `tableRoundTripDone` guard that skips `InsertContent`, preventing text around the table from being processed
- Uses fragile table-finding via selection range overlap with `GetAllTables()`
- Doesn't identify which table each cell belongs to (ambiguous with multiple tables)

**The new architecture:**
- Selection `P1 - T1 - P2` → Replace produces `P1' - T1' - P2'`, Insert produces `P1 - T1 - P2 - P1' - T1' - P2'`
- T1' is a **clone** of T1 (via `ApiTable.Copy()`) with cells modified from LLM response
- The clone is part of the `content[]` array passed to `InsertContent` alongside paragraphs
- No more `tableRoundTripDone` guard — everything goes through `InsertContent`

</domain>

<decisions>
## Implementation Decisions

### Marker contract change
- Add `[TABLE:N]...[/TABLE]` wrapper markers around cell markers in extraction
- N is the index (0-based) of the table within the selection, in document order
- Cell markers remain `[CELL:r,c]...[/CELL]` (unchanged)
- Example extraction output:
  ```
  du texte avant
  [TABLE:0]
  [CELL:0,0]contenu cellule[/CELL]
  [CELL:0,1]autre cellule[/CELL]
  [/TABLE]
  du texte après
  ```

### Table identification
- During extraction, tables are indexed in order of appearance (0, 1, 2...)
- The same order is used to match `[TABLE:N]` markers to OO tables during reinjection
- Tables are collected from the selection BEFORE InsertContent destroys the selection (same pattern as image pre-cache)

### Clone + modify approach
- `ApiTable.Copy()` creates a deep copy preserving all structure (borders, widths, backgrounds, merged cells, styles)
- Cell content of the clone is modified: `cell.Clear()` + rebuild with LLM runs + source font/size
- The clone (not the original) is added to `content[]` and passed to `InsertContent`

### InsertContent integration
- `InsertContent` accepts mixed `[ApiParagraph, ApiTable, ApiParagraph]` arrays (confirmed in OO source)
- Remove `tableRoundTripDone` guard and the separate in-place modification path
- Tables flow through the same `content[]` building loop as paragraphs

### Extraction changes needed
- `extractTableCells()` must wrap output in `[TABLE:N]...[/TABLE]`
- The table index N must be tracked during the extraction loop

### Plugin-side (React) changes needed
- `parseCellMarkers()` in `tableCellMarkers.js` must handle `[TABLE:N]` wrappers
- `transformCellMarkersForPreview()` must handle multiple tables
- System prompt augmentation must mention `[TABLE:N]` markers

### callCommand changes needed
- Pre-cache: collect tables from selection (ordered), clone each via `Copy()`
- Parse `[TABLE:N]` from markdown to know where tables go in the content flow
- During content building: when encountering a table block, modify the clone's cells and push it to `content[]`
- Remove `tableRoundTripDone` guard and in-place modification code

### What to preserve from Phase 24
- Cell content rebuilding logic (Clear + CreateRun + font/size) — reuse as-is
- `tableCellMarkers.js` utility functions — update, don't rewrite
- System prompt augmentation pattern — update markers description
- Cell validation (count mismatch warning) — update to work per-table

### Claude's Discretion
- Exact parsing implementation details
- How to represent table blocks in the `flattenTokens` output
- Error handling for edge cases (missing table, cell count mismatch per table)

</decisions>

<specifics>
## Specific Ideas

### Content building flow (pseudo-code)
```
// Before InsertContent:
var tables = collectTablesFromSelection(doc);  // ordered list of ApiTable
var clones = tables.map(t => t.Copy());        // deep copies

// Parse markdown blocks:
for each block in flattenedTokens:
  if block is paragraph/heading/list:
    content.push(buildParagraph(block))
  if block is table_placeholder(N):
    modifyCellsOfClone(clones[N], parsedCells[N])
    content.push(clones[N])

// Single InsertContent call:
doc.InsertContent(content, isInline)
```

### Extraction output example (multi-table)
```
Paragraph before first table
[TABLE:0]
[CELL:0,0]Header 1[/CELL]
[CELL:0,1]Header 2[/CELL]
[CELL:1,0]Data 1[/CELL]
[CELL:1,1]Data 2[/CELL]
[/TABLE]
Paragraph between tables
[TABLE:1]
[CELL:0,0]Other table[/CELL]
[/TABLE]
Paragraph after
```

</specifics>
