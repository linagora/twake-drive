# Phase v3.2-01 Plan 01: Discreet « Inclure » Zone (static UX) Summary

A discreet « Inclure » zone now sits above the Scribe prompt with three muted cozy-ui checkboxes (document / discussion / sélection); their state lives in ScribeContext and feeds NO value into the LLM call — v3.1 prompt assembly is byte-for-byte unchanged.

## What Was Built

- **Task 1 — i18n (`Scribe.include.*`)**: added the `include` group (`label`, `document`, `discussion`, `selection`) to all 5 target locales (fr/en/de/es/it) with native (non-English-copy) DE/ES/IT values. Parity gate green.
- **Task 2 — ScribeContext booleans**: `includeDocument` (false), `includeDiscussion` (false), `includeSelection` (true) added to the provider with setters, exposed in the `value` useMemo object + deps. A `prevSelectionRef` effect auto-checks `includeSelection` on the null→selection edge (opt-OUT model). A seam comment marks the v3.2-02/03 read point. The `sendMessage`/`aiMessages` assembly is untouched (no include boolean referenced inside it).
- **Task 3 — Component + wiring (TDD)**: new `ScribeIncludeZone.jsx` (muted 11px uppercase caption + role=group checkbox row; always-mounted `aria-live` wrapper; conditional default-checked « sélection »; cozy-ui Checkbox `size="small"` padding:2; no purple recolor; no local state for the booleans). `ChatInput` mounts the zone above the chip and gates the chip on `currentSelection && includeSelection` (the × still calls `dismissSelection`). The literal-audit `SURFACE_FILES` now scans `ScribeIncludeZone.jsx` + `ChatInput.jsx`.
