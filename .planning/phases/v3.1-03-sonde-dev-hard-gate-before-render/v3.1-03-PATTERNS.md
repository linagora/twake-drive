### `scribeProbe.js` (utility, pure transform + localStorage)

**Analog:** `src/modules/views/OnlyOffice/Scribe/scribeResponse.js` (read in full)

**Module-header / purity-contract pattern** (scribeResponse.js lines 1-31): copy the docblock
shape — state the returned object shape, declare the module pure ("no React, no network, no
module-level side effects"), and list security guarantees (ReDoS-safe linear regexes,
proto-pollution safety). The probe must keep the SAME purity contract; the only allowed side
effect is the two localStorage helpers (isolate them, mirror Pattern 3 below).

**Exported-const grammar pattern** (scribeResponse.js lines 40-53):
```js
export const MAX_RAW_LENGTH = 512 * 1024
export const FRAGMENT_MARKER_RE = /\{\{fragment:(\d+)\}\}/g
```
Mirror this for probe constants: `PROBE_KEY = 'SCRIBE_PROBE_v1'`, `PROBE_SCHEMA_VERSION = 1`,
`MAX_SAMPLES = 200`, `DUP_THRESHOLD`, and the `PREAMBLE_PATTERNS` table. Keep regexes linear /
anchored exactly like FRAGMENT_MARKER_RE (fixed literal + `\d+`, never greedy `.+?`).

**Marker-grammar REUSE — do NOT re-derive** (scribeResponse.js lines 67-79, `extractChannelMarkers`):
the probe must consume parse output, not re-parse. Reuse the EXACT marker grammars already in this
file when scanning fragments/discussion for `[TABLE:\d+]`, `[/TABLE]`, `[CELL:\d+,\d+]`, `[/CELL]`
— see `hasUnbalancedStructureMarkers` (lines 186-192) as the literal source of the intra-fragment
balance check the probe extends:
```js
function hasUnbalancedStructureMarkers(str) {
  const openTable = (str.match(/\[TABLE:\d+\]/g) || []).length
  const closeTable = (str.match(/\[\/TABLE\]/g) || []).length
  const openCell = (str.match(/\[CELL:\d+,\d+\]/g) || []).length
  const closeCell = (str.match(/\[\/CELL\]/g) || []).length
  return openTable !== closeTable || openCell !== closeCell
}
```
The probe's `hasSplitTable(parsed)` ADDS the cross-channel cases (markers in `discussion`; opens
in one fragment / closes in another) on top of this — see RESEARCH Pitfall 2.

**Pure-function-with-coercion-guards pattern** (scribeResponse.js lines 234-274): every metric
function takes possibly-untrusted input and guards types defensively before work
(`typeof raw === 'string' ? raw : String(raw)`, `Array.isArray(parsed.fragments) ? ... : []`).
Copy this guarding into `computeSampleMetrics(parsed, inputMd)` and each metric.

**Never-throw / try-catch-fallback pattern** (scribeResponse.js lines 242, 319-321 and
`buildFallback` 204-223): wrap the localStorage read/parse so a corrupt corpus resets instead of
crashing the dev session (RESEARCH Pitfall 5). Mirror the `} catch (e) { return <safe default> }`
shape.

**REF grammar** for `refIntegrity` is `\{\{REF:(scribe-ref-\d+):[^}]*\}\}` — anchored on the
`scribe-ref-N` IDENTIFIER only (visible text is allowed to change). `[VERIFIED:
plugins/onlyoffice-scribe/scripts/code.js:179 per RESEARCH]`. Do not flag REF as broken on
visible-text edits (RESEARCH Pitfall 3).

function loadDevPanelPrefs() {
  try {
    var stored = localStorage.getItem(DEV_PANELS_STORAGE_KEY)
    if (stored) {
      var parsed = JSON.parse(stored)
      return Object.assign({}, DEV_PANEL_DEFAULTS, parsed)
    }
  } catch { /* ignore */ }
  return Object.assign({}, DEV_PANEL_DEFAULTS)
}
function saveDevPanelPrefs(prefs) {
  try { localStorage.setItem(DEV_PANELS_STORAGE_KEY, JSON.stringify(prefs)) }
  catch { /* ignore */ }
}

## Shared Patterns

### Dev-mode gate (zero production cost)
**Source:** `src/.../Scribe/scribeDevMode.js` lines 16-25 (`isScribeDevMd()`).
**Apply to:** every probe capture call (ScribePopover, ScribeContext) AND the new panel render.
```js
export function isScribeDevMd() {
  try {
    return typeof window !== 'undefined' && localStorage.getItem('SCRIBE_DEV_MD') === 'true'
  } catch { return false }
}
```
Never record samples in production — always wrap `recordProbeSample` in `if (isScribeDevMd())`.

### localStorage envelope (try/catch, merge-with-defaults, version-reset)
**Source:** `ScribeResultPanel.jsx` lines 34-50 (`loadDevPanelPrefs`/`saveDevPanelPrefs`).
**Apply to:** `scribeProbe.js` corpus read/write (`SCRIBE_PROBE_v1`). Add a `v !== SCHEMA_VERSION
→ reset` check and a `MAX_SAMPLES` ring-buffer trim before write (RESEARCH Pitfall 5). Same
`try { ... } catch { /* ignore */ }` shape.

### Lazy dev chunks (highlight.js / js-beautify)
**Source:** `scribeDevMode.js` lines 34-137 (`loadBeautify`, `loadHighlightJs`) + their use in
`ScribeResultPanel.jsx` lines 327-363.
**Apply to:** the new "parsed response" panel (pretty-print + syntax-color the parsed JSON). Reuse
`loadHighlightJs()` / `loadBeautify()`; do NOT add a new loader.

### Zero-dependency pure-module contract
**Source:** `scribeResponse.js` lines 1-31 (purity + security docblock).
**Apply to:** `scribeProbe.js`. No new npm deps (RESEARCH Standard Stack: hand-rolled Dice
coefficient, not `string-similarity`). All regexes linear/anchored.

### Marker grammars (single source of truth — reuse, don't re-derive)
**Source:** `scribeResponse.js` — `FRAGMENT_MARKER_RE` (line 53), `extractChannelMarkers`
(67-79), `hasUnbalancedStructureMarkers` (186-192); REF grammar from `code.js:179`.
**Apply to:** `hasSplitTable` and `refIntegrity` in `scribeProbe.js`. Consume parse output; the
probe "n'invente pas de parsing" (CONTEXT canonical-refs).
