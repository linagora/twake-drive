# Phase v3.1-03 Plan 01: Pure conformance probe + versioned corpus Summary

**scribeProbe.js — five zero-dependency conformance metrics (word-bigram Dice duplication, per-locale preamble, cross-channel split-table, REF/footnote id set-diff, fragCount) plus a versioned, self-healing localStorage corpus with LLM-free idempotent replay and rate/count/coverage aggregation, semantics locked by 41 Jest tests.**

## Performance

- **Duration:** ~12 min
- **Completed:** 2026-06-17
- **Tasks:** 2 (both TDD)

## Accomplishments
- Five pure, ReDoS-safe metric functions over the already-parsed Scribe payload — the probe consumes parse output and never re-parses raw text
- Versioned `SCRIBE_PROBE_v1` localStorage corpus: record (ring-buffer trim to 200), export/import (round-trip + reject bad shape without mutating), replay (LLM-free, idempotent vs record-time), aggregate (D-04 rates, D-03 counts, D-05 fragDist, D-09 coverage)
- 41 Jest tests lock every metric's semantics before GATE.md numbers are frozen
- Zero new npm dependencies (hand-rolled Dice; package.json untouched)

## Decisions Made
- **One module for both tasks:** Tasks 1 and 2 both target `scribeProbe.js`; combined them into one cohesive pure module (RESEARCH §Recommended Module Structure) with a clear metrics section then persistence section.
- **No separate `scribeProbeLocales.js`:** `PREAMBLE_PATTERNS` lives inline in `scribeProbe.js` as an exported const (RESEARCH allowed "or const inside scribeProbe.js"), keeping the module self-contained.
- **Footnote ids in the REF set-diff:** `[^scribe-fn-N]` folded into the same identifier comparison as `scribe-ref-N` (Open Question 3 RESOLVED) — one extra linear regex, zero extra surface.
- **`[ASSUMED]` thresholds preserved:** `DUP_THRESHOLD=0.6` and `MAX_SAMPLES=200` are documented as starting points; the hybrid gate (D-01) freezes real numbers in GATE.md.

## Threat Surface Notes
All four `mitigate` dispositions from the plan's threat register are implemented:
- **T-v3.1-03-01 (importCorpus tampering):** version + `Array.isArray(samples)` validated before `writeCorpus`; throws on mismatch without mutating storage; JSON.parse only, no eval/Function. Test: "importCorpus throws on a shape/version mismatch WITHOUT mutating storage".
- **T-v3.1-03-02 (localStorage DoS):** ring-buffer trim to MAX_SAMPLES + write wrapped in try/catch for QuotaExceededError; corrupt/version-stale corpus resets to empty on read. Tests: version-reset, corrupt-reset, ring-buffer.
- **T-v3.1-03-03 (regex ReDoS):** all regexes linear/anchored (fixed literal + `\d+`, no nested quantifiers).
- **T-v3.1-03-SC (package legitimacy):** zero external packages installed; hand-rolled Dice; package.json unchanged.
