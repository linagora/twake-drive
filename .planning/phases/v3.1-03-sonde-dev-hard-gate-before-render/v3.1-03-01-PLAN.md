<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| imported JSON file -> importCorpus | Untrusted JSON (corpus replay file) parsed into app memory and written to localStorage |
| localStorage <-> probe module | Persisted corpus may be corrupt, oversized, or version-stale across reloads |
| LLM raw text -> metric functions | Untrusted model output (already parsed) scanned by regex metrics |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-v3.1-03-01 | Tampering | importCorpus(json) | mitigate | Validate `v === PROBE_SCHEMA_VERSION` AND `Array.isArray(samples)` before replacing storage; throw on mismatch WITHOUT mutating existing corpus (Pitfall 5). No `eval`/Function — `JSON.parse` only. |
| T-v3.1-03-02 | Denial of Service | localStorage write | mitigate | Ring-buffer trim to MAX_SAMPLES=200 + write wrapped in try/catch for QuotaExceededError; corrupt/oversized corpus resets to empty envelope on read, never crashes the dev session. |
| T-v3.1-03-03 | Denial of Service | regex metrics (Dice/preamble/REF/table) | mitigate | All regexes linear/anchored (no nested quantifiers), mirroring scribeResponse.js ReDoS-safe grammar; input already capped by parser at MAX_RAW_LENGTH=512KB. |
| T-v3.1-03-04 | Information Disclosure | corpus persistence | accept | Dev-only tool gated behind isScribeDevMd() (capture wired in Plan 02); no secrets/PII stored beyond document text the dev is already editing; never recorded in production. |
