## 1. Gate model (D-01 — hybrid)

The gate is **hybrid**:

- The **probe** (`scribeProbe.js`, dev-only) computes each conformance metric over an accumulated
  sample and raises **flags against the documented thresholds in §2** for objective failures.
- The **user pronounces the final go/no-go** (§6) after reviewing an evidence sample (§7).

Thresholds catch the objective defects (broken REF, split table, systemic duplication, systemic
preamble leakage). Human judgment covers the residual subjective quality that no numeric threshold
can capture. This is the appropriate model for a one-shot gate against a non-deterministic generic
model.

**Every numeric value in this document is tagged `[ASSUMED]` until the user freezes it against
real observed data.** The "Frozen value" column is intentionally left blank for the user to fill.

**Rationale (from RESEARCH "Proposed initial numeric values"):**
- Dice `0.6`: above ~0.6 bigram overlap usually means substantive copy, not shared stock phrases.
- `>15%` rate on the two quantitative blockers tolerates an isolated non-deterministic miss while
  flagging systemic copying / preamble leakage.
- `0` tolerance on split-table and broken-REF: these are structural / data-corruption failures.
- Coverage `>=30` total makes a 15% rate threshold statistically meaningful; per-locale `>=3` is the
  minimum to observe per-locale preamble behavior; `>=5` table and `>=5` REF cases force real
  exposure of the two hard-blocker surfaces.

These knobs map directly to `scribeProbe.js` constants and `aggregate()` outputs
(`DUP_THRESHOLD`, `dupRate`, `preambleRate`, `splitTableCount`, `refBrokenCount`, `coverage`).

## 3. Hard blockers (D-03)

Four metrics are **hard blockers** — a breach (per the rule in §2) is a NO-GO regardless of the
subjective human judgment:

1. **Broken REF / cross-refs** (D-03.1) — a `scribe-ref-N` (or `[^scribe-fn-N]` footnote)
   identifier is dropped from the output or fabricated. Identifier corruption = data corruption.
   **Zero tolerance** (threshold #5). Only the identifier is immutable; visible text may legitimately
   change (translation/edit) and is NOT counted as a breach.
2. **Split table** (D-03.2) — a table that should stay whole is cut between `discussion` and a
   fragment, or spread across fragments, or any `[TABLE:|[/TABLE]|[CELL:|[/CELL]` marker leaks into
   `discussion` (markers must live inside fragments only). **Zero tolerance** (threshold #4).
3. **Duplication discussion↔fragment** (D-03.3, quantitative D-04) — a fragment recopies all/part of
   the discussion. Measured per-response by word-bigram Dice; NO-GO is **rate-based** (threshold #2).
4. **Preamble in fragment** (D-03.4, quantitative D-04) — a fragment opens with a meta-referential
   preamble ("Voici le texte…", "Here is…", per locale) instead of pure content. NO-GO is
   **rate-based** (threshold #3).

## 4. Indicative metrics (D-05)

These are **descriptive** — displayed in the probe panel, NOT blocking:

- **Fragment distribution 0 / 1 / N** — the count of responses with 0, 1, or 2+ fragments. Useful to
  read how the model behaves per surface (popover should trend to 1; chat 0..N), but it never on its
  own forces a verdict.

**RETIRED metric (D-06):** The **A/B prose-quality comparison** ("comparaison A/B qualité de prose
vs ancien chemin plain", originally ROADMAP criterion 2) is **RETIRED** and intentionally NOT
implemented. Reasons: (a) no stable "old level" baseline exists to compare against — the old plain
path never measured or froze any prose-quality level, so the target is a moving, non-deterministic
one; (b) the real, concrete degradations (preamble, duplication, split table, broken REF) are already
covered by the objective hard blockers in §3. Building baseline machinery for an advisory, ill-defined
metric is not worth the cost. Side benefit: corpus replay (D-08) is LLM-free, because every surviving
metric is a pure function of stored text.
