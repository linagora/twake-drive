# Phase v3.1-03: Sonde dev (HARD GATE before render) - Research

**Researched:** 2026-06-17
**Domain:** Dev-only conformance probe (pure JS metrics over parsed LLM output) + DevPanelGrid extension + localStorage corpus persistence/replay
**Confidence:** HIGH (entire phase is internal to a well-mapped codebase; near-zero external dependency surface)

## Summary

This phase builds a **measurement bench**, not a feature. It reads the already-parsed
`{discussion, fragments, valid, fellBack, warnings}` produced by `parseScribeResponse`
(v3.1-01), computes conformance metrics over an accumulated sample, persists the sample to
`localStorage` with JSON export/import for offline replay, and surfaces everything in new
panels of the existing `DevPanelGrid`. The metrics feed a hybrid go/no-go gate whose
thresholds and verdict are recorded in a new `v3.1-03-GATE.md`.

The single most important architectural fact: **all remaining metrics are pure functions of
stored text** (the A/B prose-quality metric is retired per D-06), so the entire metrics layer
can — and should — be a dependency-free, framework-free pure module (`scribeProbe.js`),
mirroring the established zero-dep style of `scribeResponse.js`. This makes the corpus replay
(D-08) trivially correct: re-importing stored responses and recomputing metrics needs no LLM,
no network, no React. It also makes every metric unit-testable in Jest, which is the right way
to lock metric semantics before the GATE.md numbers are frozen.

The probe observes **two surfaces** (D-11): the inline popover (already wired to `setDevData`
in `ScribePopover.handleActionSelect`) and chat (`ScribeContext.sendMessage`, where no devData
flow exists yet and must be created). Both already call `parseScribeResponse` and already hold
the `parsed` object at the exact point where the probe must capture it.

**Primary recommendation:** Build one pure module `scribeProbe.js` (metrics + corpus
persistence, zero deps, fully Jest-tested) + extend `DevPanelGrid` with two panels (parsed
response, conformance+coverage). Capture samples by calling `recordProbeSample(parsed, meta)`
at the two existing post-parse points. Use a hand-rolled token/shingle Dice-coefficient for
duplication (no new npm dep). Detect "broken REF" by set-diffing `scribe-ref-N` identifiers
between the document input (`devData.md`) and the union of fragment bodies. Propose initial
threshold/coverage numbers in RESEARCH; the user freezes them in GATE.md against real data.

### Pattern 1: Pure metric module mirroring scribeResponse.js
**What:** Every metric is an exported pure function `(parsed, inputMd) → value`. No React, no
network, no module-level side effects except the two storage helpers.
**When to use:** Always — it is the project's established contract-layer pattern and the only way
to satisfy D-08 replay (recompute over stored text without the LLM) and to unit-test thresholds.
**Example:**
```js
// Source: pattern mirrors src/.../Scribe/scribeResponse.js (in-repo)
export function computeSampleMetrics(parsed, inputMd) {
  const frags = Array.isArray(parsed.fragments) ? parsed.fragments : []
  return {
    fragCount: frags.length,                                  // 0/1/N bucket source (D-05)
    duplication: maxFragmentDuplication(parsed.discussion, frags), // 0..1 (D-04)
    preamble: frags.some(f => detectPreamble(f) !== null),    // boolean (D-04)
    splitTable: hasSplitTable(parsed),                        // boolean (D-03.2)
    refBroken: refIntegrity(inputMd, frags).broken,          // boolean (D-03.1)
  }
}
```

### Pattern 3: Storage-versioned localStorage with quota guard
**What:** A namespaced, version-stamped envelope `{ v: 1, samples: [...] }`, written through a

**When to use:** D-08 persistence; matches the `try/catch { /* ignore */ }` style of

### Anti-Patterns to Avoid
- **Re-parsing raw text in the probe:** the probe must consume `parseScribeResponse` output, not
  re-derive it (CONTEXT canonical-refs: "elle n'invente pas de parsing").
- **Calling the LLM during replay:** D-06/D-08 — replay is LLM-free by design. Any metric that
  needs the LLM is out of scope (this is exactly why A/B was retired).
- **Putting metric logic in the React component:** it would be untestable and un-replayable. Keep
  components dumb; all logic in `scribeProbe.js`.
- **Recording samples in production:** always gate behind `isScribeDevMd()`.
- **Treating `split-table` warning from parse as the whole story:** `parseScribeResponse` only
  flags *intra-fragment* unbalanced TABLE/CELL markers. The probe's D-03.2 "table split between
  discussion and fragment" is a *broader* check (see Pitfall 2).

## Common Pitfalls

### Pitfall 1: Duplication metric over-/under-counting on a non-deterministic model
**What goes wrong:** A naive char-by-char or exact-substring duplication check either misses
near-duplicates (model rephrases slightly) or false-positives on shared short stock phrases.
**Why it happens:** LLM duplication is *semantic copy with minor edits*, not byte-identical.
**How to avoid:** Use a normalized token/shingle overlap (Dice coefficient on word bigrams or
trigrams) producing a 0..1 score; threshold per-response, then compute the *rate* of responses
over threshold (D-04). Normalize first: lowercase, strip markdown punctuation/markers, collapse
whitespace, drop `{{...}}` markers so they do not inflate overlap. Document the chosen n-gram size
and per-response threshold so GATE.md can freeze the *rate* threshold.
**Warning signs:** Duplication rate is either ~0% (too strict) or ~100% (too loose) across a
varied sample — both indicate a bad threshold, not a model verdict.

### Pitfall 2: "Split table" — the parser only sees half of it
**What goes wrong:** Relying solely on the parser's `split-table` warning misses D-03.2's real
case: a table that should stay whole gets *cut between discussion and fragment* (e.g. opening
`[TABLE:0]` ends up in `discussion`, the rest in a fragment, or rows are scattered across
fragments).
**Why it happens:** `parseScribeResponse.hasUnbalancedStructureMarkers` runs per-fragment only
(`fragments.some(...)`), and structure markers are spec'd to live *inside fragments, never in
discussion*. So a table marker appearing in `discussion` at all is already a contract violation
the parser does not flag as `split-table`.
**How to avoid:** Probe's `hasSplitTable(parsed)` should flag if ANY of:
(a) parser already set the `split-table` warning (intra-fragment imbalance), OR
(b) any `[TABLE:|[/TABLE]|[CELL:|[/CELL]` marker appears in `discussion` (markers must never be
there per SCRIBE_OUTPUT_SCHEMA), OR
(c) `[TABLE:N]` opens in one fragment and its `[/TABLE]` is absent from that same fragment but a
matching close exists in a *different* fragment (table spread across fragments). Count opens/closes
per-fragment and globally; mismatch between per-fragment balance and global balance ⇒ split.
Reuse the exact marker grammar from `scribeResponse.js` (`[TABLE:\d+]`, `[/TABLE]`, `[CELL:\d+,\d+]`,
`[/CELL]`) — linear regexes only.
**Warning signs:** Table markers showing up in the parsed-response panel's `discussion` field.

### Pitfall 3: REF detection — "broken" vs legitimately-edited visible text
**What goes wrong:** Flagging a REF as broken because its *visible text* changed (translation/edit
is explicitly allowed per the system prompt).
**Why it happens:** The marker is `{{REF:scribe-ref-N:visible text}}` — only the `scribe-ref-N`
*identifier* is immutable; the visible text may change.
**How to avoid:** Extract only the identifier set from input vs output:
`/\{\{REF:(scribe-ref-\d+):[^}]*\}\}/g` (matches the code.js grammar). Compare the *set of
`scribe-ref-N` ids* present in the document input (`devData.md` / selection markdown) against the
set present across all fragment bodies. `broken = true` if any input id is missing from output OR
an output id was never in the input (fabricated id). Ignore visible-text differences entirely.
Footnote markers `[^scribe-fn-N]` follow the same pattern (`/\[\^(scribe-fn-\d+)\]/g`) — optionally
apply the same set-diff. `[VERIFIED: plugins/onlyoffice-scribe/scripts/code.js:179 marker regex]`
**Warning signs:** REF "broken" rate spikes whenever a translate action runs — means you are
comparing visible text, not ids.

### Pitfall 4: Preamble false positives on legitimate content
**What goes wrong:** A fragment that legitimately *is about* an introduction starts with "Voici"
and gets flagged.
**Why it happens:** Preamble patterns are heuristic, not semantic.
**How to avoid:** Anchor patterns at the START of the fragment (`^\s*`), require the
meta-referential shape ("here is the/your <text/translation/summary/rewrite>" + optional colon),
and keep the set tight + documented + locale-tagged (see below). Make the pattern table a single
exported const so it is reviewable and testable. Record *which* pattern matched (for the panel).
**Warning signs:** Preamble rate near 100% — patterns too broad.

### Pitfall 5: localStorage quota / JSON.parse on corrupt corpus
**What goes wrong:** Accumulated samples exceed the ~5MB localStorage quota, or an import of a
malformed JSON file throws and wipes the dev session.
**Why it happens:** Samples store full `raw`/`discussion`/`fragments` text; many table responses
are large.
**How to avoid:** (a) Version-stamp the envelope (`{v:1,...}`) and on read, if version mismatches,
reset rather than crash. (b) Wrap all storage in try/catch like the prefs helpers. (c) On
`QuotaExceededError`, drop oldest samples (ring buffer, cap N e.g. 200) and surface a panel
warning. (d) Bound per-sample stored text (the parser already caps raw at 512KB; the probe can
truncate stored `raw` further for the corpus). (e) Import validates shape before replacing.
**Warning signs:** Silent loss of samples after long dev sessions; import does nothing.

### Per-locale preamble pattern set (proposed, DOCUMENTED per D-06b)
```js
// scribeProbe.js — anchored at fragment start; identifier kept for panel display.
// Each entry: meta-referential opener ("here is the …", "voici le …", …).
const PREAMBLE_PATTERNS = [
  // fr
  { locale: 'fr', re: /^\s*(voici|voilà)\s+(le|la|les|votre|une?)\b/i },
  { locale: 'fr', re: /^\s*je vous (propose|présente)\b/i },
  // en
  { locale: 'en', re: /^\s*here(?:'s| is| are)\s+(the|your|a|an)\b/i },
  { locale: 'en', re: /^\s*sure[,!]?\s+here\b/i },
  { locale: 'en', re: /^\s*(below is|i('ve| have) (written|prepared))\b/i },
  // de
  { locale: 'de', re: /^\s*hier (ist|sind|ist ihr|haben sie)\b/i },
  { locale: 'de', re: /^\s*(nachfolgend|im folgenden)\b/i },
  // es
  { locale: 'es', re: /^\s*(aquí (tienes|está|están)|este es|esta es)\b/i },
  // it
  { locale: 'it', re: /^\s*(ecco (il|la|i|le|un|una)|di seguito)\b/i },
]
export function detectPreamble(fragment) {
  const s = typeof fragment === 'string' ? fragment : ''
  for (const p of PREAMBLE_PATTERNS) if (p.re.test(s)) return p.locale
  return null  // null = no preamble
}
```
`[ASSUMED]` — pattern set is a documented starting point per D-06b; the *exact* phrase coverage
should be refined against the real curated corpus and confirmed during planning/discuss. All
regexes are linear (anchored alternations, no nested quantifiers) — ReDoS-safe.

### Token Dice-coefficient duplication (zero-dep)
```js
// scribeProbe.js — normalized word-bigram Dice coefficient, 0..1.
function normalizeForDup(s) {
  return (s || '')
    .toLowerCase()
    .replace(/\{\{[^}]*\}\}/g, ' ')          // drop fragment/REF markers
    .replace(/\[[^\]]*\]/g, ' ')             // drop [TABLE:..]/[CELL:..] markers
    .replace(/[^\p{L}\p{N}\s]/gu, ' ')       // strip punctuation
    .replace(/\s+/g, ' ').trim()
}
function bigrams(tokens) {
  const out = new Map()
  for (let i = 0; i < tokens.length - 1; i++) {
    const g = tokens[i] + ' ' + tokens[i + 1]
    out.set(g, (out.get(g) || 0) + 1)
  }
  return out
}
function diceCoefficient(a, b) {
  const A = bigrams(normalizeForDup(a).split(' ').filter(Boolean))
  const B = bigrams(normalizeForDup(b).split(' ').filter(Boolean))
  if (A.size === 0 || B.size === 0) return 0
  let inter = 0
  for (const [g, ca] of A) { const cb = B.get(g); if (cb) inter += Math.min(ca, cb) }
  const total = [...A.values()].reduce((x,y)=>x+y,0) + [...B.values()].reduce((x,y)=>x+y,0)
  return (2 * inter) / total
}
// Per-response duplication = max overlap between discussion and any fragment.
export function maxFragmentDuplication(discussion, fragments) {
  return (fragments || []).reduce((m, f) => Math.max(m, diceCoefficient(discussion, f)), 0)
}
```

### REF integrity by identifier set-diff

```js
// scribeProbe.js
const REF_ID_RE = /\{\{REF:(scribe-ref-\d+):[^}]*\}\}/g   // matches code.js grammar
function refIds(text) {
  const ids = new Set(); let m
  const re = new RegExp(REF_ID_RE.source, 'g')
  while ((m = re.exec(text || '')) !== null) ids.add(m[1])
  return ids
}
export function refIntegrity(inputMd, fragments) {
  const inIds = refIds(inputMd)
  const outIds = new Set()
  for (const f of (fragments || [])) for (const id of refIds(f)) outIds.add(id)
  const missing = [...inIds].filter(id => !outIds.has(id))   // dropped REF = corruption
  const fabricated = [...outIds].filter(id => !inIds.has(id)) // invented REF
  return { broken: missing.length > 0 || fabricated.length > 0, missing, fabricated }
}
```

### Versioned localStorage corpus + export/import/replay
```js
// scribeProbe.js — mirrors loadDevPanelPrefs/saveDevPanelPrefs try/catch style.
const PROBE_KEY = 'SCRIBE_PROBE_v1'
const PROBE_SCHEMA_VERSION = 1
const MAX_SAMPLES = 200

function readCorpus() {
  try {
    const raw = localStorage.getItem(PROBE_KEY)
    if (!raw) return { v: PROBE_SCHEMA_VERSION, samples: [] }
    const parsed = JSON.parse(raw)
    if (!parsed || parsed.v !== PROBE_SCHEMA_VERSION || !Array.isArray(parsed.samples))
      return { v: PROBE_SCHEMA_VERSION, samples: [] }   // reset on version/shape mismatch
    return parsed
  } catch { return { v: PROBE_SCHEMA_VERSION, samples: [] } }
}
function writeCorpus(corpus) {
  try { localStorage.setItem(PROBE_KEY, JSON.stringify(corpus)) }
  catch { /* QuotaExceededError: caller already trimmed to MAX_SAMPLES */ }
}
export function recordProbeSample(parsed, meta) {
  const corpus = readCorpus()
  corpus.samples.push({
    ts: meta.ts || Date.now(),
    surface: meta.surface,           // 'popover' | 'chat'
    inputMd: meta.inputMd || '',     // for REF/dup input comparison + replay
    discussion: parsed.discussion,
    fragments: parsed.fragments,
    valid: parsed.valid,
    fellBack: parsed.fellBack,
    warnings: parsed.warnings,
    // optional manual tags (D-07/D-09 coverage): locale, hasTable, hasRef
    tags: meta.tags || {}
  })
  if (corpus.samples.length > MAX_SAMPLES) corpus.samples.splice(0, corpus.samples.length - MAX_SAMPLES)
  writeCorpus(corpus)
  return corpus.samples.length
}
export function exportCorpus() { return JSON.stringify(readCorpus(), null, 2) }
export function importCorpus(json) {            // replace; validate shape
  const parsed = JSON.parse(json)
  if (!parsed || parsed.v !== PROBE_SCHEMA_VERSION || !Array.isArray(parsed.samples))
    throw new Error('probe-corpus: unsupported shape/version')
  writeCorpus(parsed)
  return parsed.samples.length
}
// Replay = recompute metrics over stored samples, NO LLM:
export function replay(corpus) {
  const c = corpus || readCorpus()
  return c.samples.map(s => ({ ...s, metrics: computeSampleMetrics(s, s.inputMd) }))
}
```
