<problem>
The current implementation attaches `mousemove` and `mouseup` listeners to `document`. When the pointer moves over the OnlyOffice iframe (to the left of the panel, which is exactly where the user drags to widen the panel), the iframe captures mouse events and the document-level listeners stop firing. Result: the drag freezes as soon as the pointer crosses into the iframe area.

**Root cause** (ResizeHandle.jsx:16-39):
- `onMouseDown` calls `document.addEventListener('mousemove', onMouseMove)` and `document.addEventListener('mouseup', onMouseUp)`
- Cross-origin iframes (OO editor is served from a different origin) intercept pointer events when the cursor is over them, so the parent document never sees the mousemove
</problem>

<solution>
Use the **Pointer Events API** with `setPointerCapture` on the handle element. Once pointer capture is set, all subsequent pointer events for that pointer ID are routed to the capturing element regardless of where the pointer physically moves — including over iframes.

This is the standard pattern for drag handles next to iframes (e.g. VSCode split panels, CodeSandbox, Figma).
</solution>
</context>

  <action>
Rewrite the drag logic in `ResizeHandle.jsx` to use Pointer Events instead of Mouse Events:

1. **Remove** the `onMouseMove`, `onMouseUp`, `onMouseDown` callbacks that use `document.addEventListener`.

2. **Add** three handlers attached directly to the handle `<div>`:
   - `onPointerDown(e)`:
     - `e.preventDefault()`
     - `e.currentTarget.setPointerCapture(e.pointerId)` — critical line, routes all subsequent events for this pointer to the handle element even when over iframes
     - Store `pointerId` in a ref for later release
     - Set `isDragging` + `isDraggingRef.current = true`
     - Set `document.body.style.userSelect = 'none'` and `document.body.style.cursor = 'col-resize'`
   - `onPointerMove(e)`:
     - If `!isDraggingRef.current` return
     - Compute `const newWidth = window.innerWidth - e.clientX`
     - Call `setPanelWidth(newWidth)`
   - `onPointerUp(e)` (and `onPointerCancel` mirroring it):
     - If `!isDraggingRef.current` return
     - `e.currentTarget.releasePointerCapture(e.pointerId)` (guard with try/catch in case already released)
     - Clear `isDragging`, `isDraggingRef.current = false`
     - Restore `document.body.style.userSelect = ''` and `document.body.style.cursor = ''`

3. **Update JSX** — attach the new handlers to the root `<div>`:
   ```jsx
   <div
     onPointerDown={onPointerDown}
     onPointerMove={onPointerMove}
     onPointerUp={onPointerUp}
     onPointerCancel={onPointerUp}
     onMouseEnter={() => setIsHovered(true)}
     onMouseLeave={() => setIsHovered(false)}
     style={{ /* unchanged */ }}
   >
   ```

4. **Cleanup useEffect** — keep it but simplify: only reset body styles on unmount (no document listeners to remove). Pointer capture is automatically released if the element unmounts.

5. **Touch-action** — add `touchAction: 'none'` to the handle's inline `style` to prevent browser gesture interference (scroll, zoom) on touch devices.

**Keep everything else identical**: hover/dragging visual state, accent line, theme colors, component exports.
