# v3.0-04-02 — Fix resize handle drag over iframe

## Problem

The ResizeHandle component used `document.addEventListener('mousemove', ...)` to track drag motion. When the pointer crossed into the OnlyOffice editor iframe area (which is exactly where users drag to widen the panel), the cross-origin iframe intercepted mouse events and the document-level listeners stopped firing. Result: the drag froze mid-motion.

## Solution

Replaced document-level mouse listeners with **Pointer Events API + `setPointerCapture`** on the handle element itself. Once pointer capture is set, all subsequent pointer events for that pointer ID are routed to the capturing element regardless of where the pointer physically moves — including over cross-origin iframes.

This is the standard pattern for drag handles adjacent to iframes (VSCode split panels, Figma, CodeSandbox).

## Changes

**`src/modules/views/OnlyOffice/Scribe/ResizeHandle.jsx`** (+33 / -21 lines)

- Removed `onMouseDown` / `onMouseMove` / `onMouseUp` callbacks using `document.addEventListener`
- Added `onPointerDown` (calls `setPointerCapture`, stores pointerId in ref, sets dragging state + body cursor/userSelect)
- Added `onPointerMove` (guarded by `isDraggingRef`, computes new width, calls `setPanelWidth`)
- Added `onPointerUp` (releases pointer capture in try/catch, clears dragging state, restores body styles)
- Wired `onPointerCancel` to the same handler as `onPointerUp`
- Simplified unmount `useEffect` (only resets body styles — no listeners to remove)
- Added `touchAction: 'none'` to the handle's inline style (prevents browser gesture interference on touch devices)
- Preserved all visual behavior: hover state, drag accent line, theme colors, col-resize cursor, JSX structure

## Verification

Manual test passed (2026-04-05):

1. ✓ Drag left across the iframe boundary — panel widens continuously, no freeze
2. ✓ Drag right — panel narrows smoothly
3. ✓ Release over iframe — drag ends cleanly, pointer capture released
4. ✓ Cursor resets to normal after release (no stuck col-resize)

Automated verify: PASS (setPointerCapture + releasePointerCapture + onPointer* handlers + touchAction, no document.addEventListener remaining).

## Key Technical Details

- `setPointerCapture(pointerId)` is called on `e.currentTarget` (the handle div) in `onPointerDown`
- `releasePointerCapture` wrapped in try/catch in case capture was already released (defensive)
- `pointerIdRef` stores the captured pointer ID for cleanup tracking
- Pointer capture is automatically released if the element unmounts during drag — no manual cleanup needed

## Known Limitations

None observed. The fix is minimal and targeted.
