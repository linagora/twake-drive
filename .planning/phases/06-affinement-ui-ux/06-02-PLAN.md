**mockTransform.js -- Complete refactor:**

Replace the entire switch/case body with a config-driven approach:

1. Import `SCRIBE_ACTIONS` and `FREE_PROMPT_CONFIG` from scribeActions.js.

2. Create a `findActionConfig(actionId)` helper that searches SCRIBE_ACTIONS (including children) for the matching id and returns the action entry. For 'free-prompt', return FREE_PROMPT_CONFIG. For translate-custom, return the translate-custom child entry.

3. Create a `applyMockResult(mockResult, text, extra)` function that interprets the `mockResult` string from the config:
   - `'capitalize'`: capitalize first letter of each `$ `-prefixed line
   - `'wrap:before:after'`: wrap prefixed text with before/after strings (split on `:`)
   - `'truncate-half'`: take first half of words per line, prefix with `$ `
   - `'suffix: text'`: append text after each `$ `-prefixed line
   - `'emojify'`: prefix each line with `$ ` + sparkles emoji, suffix with party emoji
   - `'bullets'`: prefix each line with `$ ` + bullet character
   - `null` or unrecognized: default `[Mock transform]\n{prefixed}\n[/Mock transform]`

   The `extra` parameter is used for translate-custom to fill `{language}` placeholders in the mockResult string.

4. The new `mockTransform(actionId, text, extra)` signature:
   - Looks up config via `findActionConfig(actionId)`
   - If not found, applies default mock
   - If found, calls `applyMockResult(config.mockResult, text, extra)`
   - The `extra` object can contain `{ language: 'Spanish' }` for translate-custom

5. Keep the MOCK_PREFIX = '$ ' constant and the capitalize helper.

**ScribePopover.jsx -- Minor update:**

The `handleActionSelect` callback currently receives `(actionId, label, breadcrumb)`. No change needed to this signature. The mockTransform call `mockTransform(actionId, selectedText)` remains the same for all actions except translate-custom.

For translate-custom, the ScribeActionMenu already calls `onSelect('translate-custom', trimmed, 'Translate > ' + trimmed)` where `trimmed` is the custom language name. Update ScribePopover's handleActionSelect to detect translate-custom and pass the language as extra:

```js
const handleActionSelect = useCallback(
  (actionId, label, breadcrumb) => {
    const extra = actionId === 'translate-custom' ? { language: label } : undefined
    const transformed = mockTransform(actionId, selectedText, extra)
    setResult({ text: transformed, breadcrumb })
    setStep('result')
  },
  [selectedText]
)
```

This is the only change in ScribePopover.jsx.

**Key constraint:** The existing behavior for all actions must remain identical after the refactor. The mock output for each action should look the same as before. This is a pure refactor -- same inputs, same outputs, different code path.

**Note on translate actions:** The old mockTransform had cases like `translate-francais`, `translate-anglais` etc. which are from an earlier hard-coded translate menu. The current dynamic translate system generates IDs like `translate-en`, `translate-fr`. The refactored version will handle ALL translate-* IDs via the config lookup in buildTranslateChildren's output, making the old hardcoded translate cases unnecessary.
