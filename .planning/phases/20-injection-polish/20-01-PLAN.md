**Spacing detection (add BEFORE the content-building loop, after srcFont detection):**

For replace mode (mode !== "insert"):
1. Get selection range via `doc.GetRangeBySelect()`
2. Read up to 5 chars before selection start: `doc.GetRange(Math.max(0, selStart - 5), selStart).GetText()`
3. Read up to 5 chars after selection end: `doc.GetRange(selEnd, selEnd + 5).GetText()`
4. Use the same WS regex as pasteHtml: `var WS = /[\s\n\r\t\u00A0]/;`
5. Set `needSpaceBefore = true` if last char before is non-whitespace and non-empty
6. Set `needSpaceAfter = true` if first char after is non-whitespace and non-empty

For insert mode:
1. Read chars around the collapsed cursor position (selEnd) using the same pattern
2. Set needSpaceBefore based on char immediately before selEnd
3. Set needSpaceAfter based on char immediately after selEnd
4. Note: the existing cursor-collapse code (lines 244-252) already runs before this -- keep that, add spacing detection after it but before the content loop

**Space run injection (add AFTER the content-building loop, BEFORE InsertContent):**

If `needSpaceBefore` and `content.length > 0`:
- Create a space run: `var spaceRun = Api.CreateRun(); spaceRun.AddText(" ");`
- Apply srcFontFamily and srcFontSize if available (so the space matches surrounding text)
- Prepend to the first paragraph's elements. Since AddElement only appends, the approach is:
  - Create a new paragraph, add the space run, then re-add all elements from the original first paragraph
  - OR simpler: build a dedicated "space paragraph" is wrong -- the space must be in the same paragraph
  - BEST approach: restructure the first-paragraph building in the content loop. Add a flag `isFirstBlock = (i === 0 && needSpaceBefore)`. When building the first paragraph, add the space run via `p.AddElement(spaceRun)` BEFORE the loop that adds content runs.

If `needSpaceAfter` and `content.length > 0`:
- Create a space run with " ", apply srcFont properties
- Append to the last paragraph: `content[content.length - 1].AddElement(spaceRunAfter)`

For insert mode paragraph separator:
- After the cursor collapse and BEFORE InsertContent, the content already gets inserted at cursor position
- For insert mode, the content should start on a new line. The simplest approach: add an empty paragraph as the first element of the content array: `content.unshift(Api.CreateParagraph())`
- Test whether this creates the visual separation. If InsertContent already creates paragraph breaks naturally, this may not be needed -- but the research recommends it.

**Important implementation details:**
- The spacing detection must happen inside the callCommand (it needs doc.GetRange), not outside
- The selRange variable is already read for srcFont detection (line 218) -- reuse it for spacing
- Guard against null ranges (document start/end edge cases)
- Keep the existing insert-mode cursor collapse (lines 244-252) unchanged
- The spacing variables (needSpaceBefore, needSpaceAfter) must be declared with `var` at function scope level inside the callCommand
