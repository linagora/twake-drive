<objective>
Add post-injection selection so the user sees injected content highlighted after Replace or Insert.

Purpose: After Scribe injects AI-modified text, the user has no visual feedback about what was injected. Selecting the injected content gives immediate confirmation. This uses an element-count approach: count paragraphs before InsertContent, count again after, then build a range from the first to the last inserted element and Select() it — all inside the same callCommand (single undo point).

Output: After injection, the injected content is highlighted in the OO editor. If the selection fails, content is still injected (graceful degradation).
</objective>

  <action>
This task adds post-injection selection inside the existing callCommand in buildAndInject. ALL code must be ES5 (var, no arrow functions, no template literals, no const/let).

The approach: count elements before InsertContent, count after, get the first and last inserted elements, build a range spanning them, and Select() it. All in the same callCommand — no second callCommand needed, preserving the single undo point.

**Modify the callCommand body inside buildAndInject:**

Inside the existing `window.Asc.plugin.callCommand(function() { ... })` in buildAndInject, make these changes:

1. **Before InsertContent**, save the element count:
   ```javascript
   var countBefore = doc.GetElementsCount();
   ```

2. **After InsertContent** (replace the existing `doc.InsertContent(content);`), add selection logic:
   ```javascript
   if (content.length > 0) {
     doc.InsertContent(content);

     // Post-injection selection: select all inserted elements
     try {
       var countAfter = doc.GetElementsCount();
       var insertedCount = countAfter - countBefore;

       if (insertedCount > 0) {
         var firstInserted = doc.GetElement(countBefore);
         var lastInserted = doc.GetElement(countAfter - 1);

         if (firstInserted && lastInserted) {
           var startRange = firstInserted.GetRange(0, 0);
           var endRange = lastInserted.GetRange();

           if (startRange && endRange) {
             var fullRange = startRange.ExpandTo(endRange);
             if (fullRange) {
               fullRange.Select();
             }
           }
         }
       }
     } catch (e) {
       // Selection failed — content is still injected, graceful degradation
     }
   }
   ```

3. **Handle replace vs insert mode count difference:**
   In replace mode, InsertContent replaces the current selection, so countAfter may not simply be countBefore + content.length (the replaced paragraphs are removed). The logic `insertedCount = countAfter - countBefore` handles this naturally:
   - If we replaced 1 paragraph with 3: countAfter = countBefore + 2, insertedCount = 2
   - But we need ALL inserted elements, not just the "extra" ones

   **Revised approach:** Instead of relying on count arithmetic, use a simpler strategy — we know exactly how many elements we're inserting (content.length). The inserted elements occupy positions from `countAfter - content.length` to `countAfter - 1`:
   ```javascript
   if (content.length > 0) {
     doc.InsertContent(content);

     // Post-injection selection
     try {
       var countAfter = doc.GetElementsCount();
       var firstIdx = countAfter - content.length;
       if (firstIdx < 0) firstIdx = 0;
       var lastIdx = countAfter - 1;

       var firstInserted = doc.GetElement(firstIdx);
       var lastInserted = doc.GetElement(lastIdx);

       if (firstInserted && lastInserted) {
         var startRange = firstInserted.GetRange(0, 0);
         var endRange = lastInserted.GetRange();

         if (startRange && endRange) {
           var fullRange = startRange.ExpandTo(endRange);
           if (fullRange) {
             fullRange.Select();
           }
         }
       }
     } catch (e) {
       // Selection failed — content still injected
     }
   }
   ```

   **Note:** This assumes InsertContent places our elements at the end of the inserted position. If testing reveals the elements are placed elsewhere (e.g. at the original selection position), adjust `firstIdx` accordingly. The countBefore approach may work better:
   ```javascript
   var firstIdx = countBefore;  // for insert mode (appended after cursor)
   ```
   The executor should test both approaches and use whichever works.

**Important notes:**
- Do NOT create a second callCommand — keep everything in the existing one for single undo
- Do NOT move `pasteInProgress = false` — it stays in the callback as-is
- The try/catch ensures that if any GetElement/GetRange/ExpandTo/Select call fails, the content is still injected
- All code must be ES5: use `var`, no arrow functions
